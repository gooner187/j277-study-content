## Network Threats – Identification and Description

### Q: Name and describe one threat to a computer system [3 marks]

```
keywords: network threats, social engineering, shoulder surfing, phishing, pharming, DOS, DDOS, denial of service, hacker, virus, malware, trojan, worm, ransomware, physical threat, describe threat
year: 2023 | paper: J277 | question: 4b | marks: 3
```

**Question:**  
Name and describe one threat to a computer system that is not: Spyware, Brute-force attack, Data interception, or SQL injection.

**Mark Scheme** (1 mark for threat name + 1 mark each for description, max 2):

**Social engineering:**
- Using deception to manipulate users
- … to gain personal data

**Shoulder surfing:**
- Watching a person entering a password
- … and using it to access an account

**Phishing:**
- Fake emails sent to person / click on link from fake email
- Person sends / gives away personal data

**Pharming:**
- Software redirects user to fake website / use of a fake website
- Person enters / gives away personal data

**Denial of Service (DOS / DDOS):**
- Multiple requests sent to a server simultaneously / server is flooded with requests
- More requests than the server can process / uses all bandwidth
- Server cannot respond / crashes / stops or slows access to a network

**Hacker:**
- Person gaining unauthorised access to a system/account
- To delete / damage / access data

**Virus / Malware:**
- Software that replicates/spreads
- Fills disk space / deletes or corrupts data / allows unauthorised access

**Trojan:**
- Malware disguised as legitimate software
- Once installed acts as a virus / by example (e.g. deleting files / unauthorised access)

**Worm:**
- Software that replicates across a network
- Uses up all the bandwidth

**Ransomware:**
- Encrypts / corrupts / locks access to data
- Cannot access data without paying a fee / meeting demands

**Physical threat:**
- Damage to hardware
- Deletes / corrupts data

**Examiner Guidance:**

- If threat name is clearly wrong, do NOT FT for description
- If no threat name given, read description; if name is in description, award description
- Social engineering is acceptable as the threat — with phishing/pharming/shoulder surfing in the description
- Ransomware: MP3 cannot be awarded for "ransom" alone — must reference paying a fee

**Question:**  
A hospital stores patients' details on its computer network.

- Rules that define acceptable use of the network

```
01  message = input("Please enter your string")
02  newMessage = ""
03  messageLength = message.length
04  for count = 0 to ___________________________
05    ASCIIValue = ASC(message.subString(_______________, 1))
06    ASCIIValue = ASCIIValue + _______________
07    if ASCIIValue > 90 then
08      ASCIIValue = _______________ – 26
09    endif
10    newMessage = _______________ + CHR(ASCIIValue)

**Mark Scheme** (1 mark per correct blank):

**Examiner Guidance:**

- Spelling must be exact; do NOT penalise case differences

## Software-Based Security Methods

### Q: Identify and describe two software-based security methods [6 marks]

```
keywords: software security, anti-malware, firewall, encryption, user access levels, anti-virus, anti-spyware, passwords, biometrics, two-step authentication, two-factor authentication
year: 2022 | paper: J277 | question: 5b | marks: 6
```

**Question:**  
Identify and describe two software-based security methods that the company can use to protect their computer systems and data.

**Mark Scheme** (1 mark for method name + up to 2 marks for description, max 6):

**Anti-malware:**
- Scans for/identifies virus/spyware/malware
- Compares data to a database of malware
- Alerts user and requests action / quarantines/deletes virus/spyware/malware
- Stops the download of virus/spyware/malware

**Firewall:**
- Scans incoming and outgoing traffic
- Compares traffic to a set of criteria
- Blocks traffic that is unauthorised / blocks incoming/outgoing traffic

**Encryption:**
- Scrambles data using an algorithm
- So if intercepted it cannot be understood
- A key is needed to decrypt

**User access levels:**
- Data can be read / write / read-write (or by example)
- Prevents accidental changes
- Limits the data users can access

**Anti-virus:**
- Scans for/identifies virus/malware
- Compares data to a database of viruses/malware
- Alerts user / quarantines/deletes virus / stops the download of virus/malware

**Anti-spyware:**
- Scans for/identifies spyware/keylogger
- Compares data to a database of spyware
- Alerts user / quarantines/deletes spyware

**Passwords / biometrics / authentication:**
- Code/fingerprint etc. has to be correctly entered to gain access
- Strong password (e.g. letters, numbers, symbols) / fingerprint is unique to individual
- Harder/impossible for a brute-force attack to succeed
- Lock after set number of failed attempts

**Two-step authentication:**
- A code is sent to the user's separate device
- Unauthorised person will also need access to this separate device

**Examiner Guidance:**

- Mark method name first; if method is wrong do NOT read on
- If second method is a repeat of the first (e.g. password then locking out), mark whole answer for max 3

```

**Question:**

**Mark Scheme** (1 mark for naming method + 1 mark for description, max 2 methods, max 4 marks):

**Password:**
- No access without the password / description of a strong password / limit login attempts / changing it regularly
- Limited attempts to guess before laptop is locked

**Firewall:**
- Monitor incoming and outgoing transmissions / stop unauthorised/unwanted incoming or outgoing transmissions/packets

- So that other people cannot physically access it

## Network Performance

*(See also [1.3 Computer Networks](./1.3-Computer_Networks.md#factors-affecting-network-performance))*

### Q: Explain how number of devices affects network performance [3 marks]

```
keywords: network performance, number of devices, bandwidth, latency, collisions, concurrent users, slow network
year: 2022 | paper: J277 | question: 3a(i) | marks: 3
```

**Question:**  
A library has a LAN. Users have reported that the network sometimes runs very slowly.  
Explain why the number of devices using the network at the same time can affect the performance of the network.

**Mark Scheme** (1 mark each, max 3):

- Slower transmission of data / less data can be transmitted at the same time
- More devices mean more data is being transmitted at a time
- Bandwidth will be split between all the devices / each device uses some of the bandwidth
- … this means there is less bandwidth available for each device
- Devices have to wait longer before they can transmit / increased latency
- If the maximum bandwidth is used, devices cannot transmit
- Central device/switch/router has to handle more requests and may run slower
- More collisions (likely) / higher error rate
- … more data has to be retransmitted

**Examiner Guidance:**

- The question asks WHY — more devices do NOT decrease the bandwidth of the network; they decrease the amount allocated/available to each device
- Do NOT accept "higher contention ratio"

## Packet Switching

### Q: Describe how packet switching is used when sending data [4 marks]

```
keywords: packet switching, packets, header, packet number, destination address, route, reassemble, order, missing packets
year: 2021 | paper: J276 | question: 7c(iii) | marks: 4
```

**Question:**  
The data transmitted between the two sites uses packet switching.  
Describe how packet switching can be used when sending data from one site to the other.

**Mark Scheme** (1 mark each, max 4):

- Data is split into packets (by the originating computer)
- Each packet has a fixed size
- Each packet has a header
- … that includes e.g. packet number / destination address
- Packets are sent individually
- Packets can be sent via different routes
