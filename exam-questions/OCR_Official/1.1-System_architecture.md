## Fetch-Execute Cycle

### Q: Describe what happens during the fetch-execute cycle [2 marks]

```
keywords: fetch execute cycle, FDE, CPU, fetch, decode, execute, registers
year: 2024 | paper: J277 | question: 6a | marks: 2
```

**Question:**  
A computer has a Central Processing Unit (CPU).  
Describe what happens during the fetch-execute cycle.

**Mark Scheme** (award 1 mark each, max 2):

- Data/instructions are fetched from memory / RAM / primary storage
- Data/instructions are stored using the registers (or correct example of a register storing address/data)
- Data/instructions are decoded / split into opcode and operand
- Data/instructions are executed / processed
- ALU performs the logical/arithmetic calculations

**Examiner Guidance:**

- MP4: BOD "carried out" etc. for "executed"
- Ignore inaccurate references to registers and components (other than MP2 correct example of a register)

## CPU Registers

### Q: Name two registers used in the fetch-execute cycle and state the purpose of each [4 marks]

```
keywords: CPU registers, program counter, PC, MAR, MDR, accumulator, ACC, CIR, memory address register, memory data register, current instruction register, fetch-execute cycle, Von Neumann
year: 2024 | paper: J277 | question: 6b | marks: 4
```

**Question:**  
Complete the table by writing the name of two registers used in the fetch-execute cycle and the purpose of each register.

**Mark Scheme** (1 mark for naming register + 1 mark for correct matching purpose):

**Program Counter (PC)**
- Stores the address of the current/next instruction to be fetched
- Stores the address of the instruction for the current/next FE cycle

**Memory Address Register (MAR)**
- Stores the address of the current/next instruction/data to be fetched
- Stores the address where data/instruction is to be stored

**Memory Data Register (MDR)**
- Stores the data/instruction fetched from memory
- Stores data/instruction to be stored in memory
- Stores the data/instruction located in the memory location in the MAR

**Accumulator (ACC)**
- Stores the result of calculations
- Stores data currently being processed (or by example)
- Stores the result from the ALU

**Current Instruction Register (CIR) / Instruction Register (IR)**
- Stores the instruction currently being executed

**Examiner Guidance:**

- Purpose must not be an action (e.g. "fetches", "takes", "retrieves" are NOT acceptable)
- Read full purpose and award a correct point
- BOD: memory buffer register for MDR
- If register column is blank but register name appears in the purpose column, award the purpose if accurate
- If the register name given is incorrect, do NOT mark the purpose
- For PC and MAR, accept "pointer" for "storing address"
- Accept "memory address" / "memory data" as register names

### Q: Describe the purpose of two registers used by Von Neumann architecture [4 marks]

```
keywords: CPU registers, Von Neumann architecture, MAR, MDR, program counter, PC, accumulator, CIR, memory address register, memory data register
year: 2018 | paper: J276 | question: 4a | marks: 4
```

**Question:**  
Alicia has designed a computer using Von Neumann architecture.  
Describe the purpose of two registers that are used by Von Neumann architecture.

**Mark Scheme** (1 mark for register name + 1 mark for purpose, max 2 registers, max 4 marks):

**MAR / Memory Address Register:**
- Stores the address/location where data will be read/written/accessed/fetched
- Stores the address/location of the data/instruction being or next to be processed

**MDR / Memory Data Register:**
- Stores the data/instruction that is fetched/read from memory
- Stores the data that is to be written to memory
- Stores the data/instruction from the address in the MAR

**Program Counter (PC):**
- Stores the address/location of the next instruction to be run
- Stores the address/location of the current instruction being run

**Accumulator:**
- Stores the result of manipulation/process/calculation

**Current Instruction Register (CIR) / Instruction Register (IR):**
- Stores the instruction currently being processed

**Examiner Guidance:**

- "MAR stores address" is NOT enough for description — must say what address
- "MDR stores the data" is NOT enough for description — must qualify what data
- Accept MBR / Memory Buffer Register for MDR

## Cache Size and CPU Performance

### Q: Explain how cache size can affect the performance of the CPU [2 marks]

```
keywords: cache size, CPU performance, cache memory, RAM, frequently used instructions, cache hit, fast memory
year: 2019 | paper: J276 | question: 1a(iii) | marks: 2
```

**Question:**  
One computer has 64 kilobytes of cache and the other has 512 kilobytes of cache.  
Explain how the cache size can affect the performance of the CPU.

**Mark Scheme** (1 mark per bullet, max 2):

- Cache stores frequently/recently/next-to-be-used instructions/data
- … that can be accessed faster than accessing them from RAM
- … which means more cache improves the performance of the CPU / less cache decreases the performance
- Too much cache can be detrimental …
- … as it will take longer to find the instructions in cache

**Examiner Guidance:**

- No mark for just defining cache as "fast memory" or "memory close to the CPU"
- No mark for "cache is faster than RAM" alone — faster at what?
- BOD "More cache makes the processing/computer faster"

## Dual-Core and Multi-Core Processors

### Q: Explain why having a dual core processor might improve the performance of the computer [2 marks]

```
keywords: dual core processor, multi-core, parallel processing, cores, performance, multitasking, single core, number of cores
year: 2019 | paper: J276 | question: 1a(ii) | marks: 2
```

**Question:**  
Kerry is looking at two computers; one has a single core processor and the other has a dual core processor.  
Explain why having a dual core processor might improve the performance of the computer.

**Mark Scheme** (1 mark per bullet, max 2):

- Dual core is 2 processors/cores / double the number of processors/cores
- Parallel processing can take place
- … each processor can execute a separate instruction at the same time / each core can process instructions independently of each other
- … which enables multitasking
- Some processes/software cannot be split between two processors so it does not increase the performance

**Examiner Guidance:**

- Must convey processors acting at the same time — "it can run twice as many instructions" without "at the same time" is NOT enough
- Do NOT award "more instructions per second" without context of simultaneity (could just mean faster clock speed)
- Allow FDE for "executing instructions"
- Do NOT allow "cores can split the tasks" — must state how (one task for each core at the same time)
- BOD "run more than one program at once"

### Q: Explain why a quad-core processor is not always twice as fast as a dual-core processor [3 marks]

```
keywords: quad core, dual core, multi-core, processor speed, parallel processing, software limitations, clock speed, cores, performance
year: 2018 | paper: J276 | question: 4c | marks: 3
```

**Question:**  
Alicia says: "My computer has a quad-core processor, so it will run twice as fast as a computer with a dual-core processor."  
Explain why this statement is not always true.

**Mark Scheme** (1 mark per bullet, max 3):

- Software may be designed to run on 1 core and not multiple cores / depends on the task(s)
- … some tasks cannot be split across cores
- Clock speed also affects speed / dual core may have a faster clock speed / quad-core may have slower clock speed
- … so one task may be run faster on fewer cores
- RAM size also affects speed / quad-core may have less RAM / amount of VM being used
- Cache size also affects speed / quad-core may have less cache

**Examiner Guidance:**

- Allow marks for other components that could affect speed (e.g. secondary storage access speed, onboard GPU)
- Award description of concurrent processing

Complete the sentences by filling in the missing words.

> CPU stands for _____________________________________________.  
> It is the part of the computer that fetches and executes the _____________

> the _____________________________ Unit (CU).

**Mark Scheme** (1 mark for each correct word/phrase):

- Central Processing Unit

## Embedded Systems

### Q: Explain the reasons why the 'Follow Me' car system is an example of an embedded system [3 marks]

```
keywords: embedded system, dedicated purpose, microprocessor, firmware, ROM, operating system, control system, automated, built-in
year: 2024 | paper: J277 | question: 7a | marks: 3
```

**Question:**  
A car has a 'Follow Me' system that uses a cruise control feature to allow the car to follow the car in front of it. It will keep the same speed and distance without the driver's intervention.  
Explain the reasons why the 'Follow Me' system is an example of an embedded system.

**Mark Scheme** (1 mark each, max 3):

- Has a specific purpose / it only performs one/limited task / dedicated to the Follow Me system
- Built within a larger device / car
- Has dedicated/specific hardware / sensors
- Has a microprocessor
- Has a built-in operating system/software / software is all in firmware/ROM
  - → its instructions/operation does not / is hard to change/update
- It is a control system / it is automated

**Examiner Guidance:**

- MP2: BOD reference to it being "built into" something reasonable

### Q: Identify one embedded system in a car and explain why it is an embedded system [3 marks]

```
keywords: embedded system, car, dedicated purpose, microprocessor, limited function, firmware, examples of embedded systems
year: 2023 | paper: J277 | question: 7 | marks: 3
```

**Question:**  
A car comes with many embedded systems, for example parking sensors.  
Identify one other embedded system that could be found in a car and explain why this is an embedded system.

**Mark Scheme:**

*1 mark for a valid example embedded system in a car (accept any reasonable answer, including):*

- Auto lights
- Auto window wipers
- Sat nav / GPS
- Air conditioning / climate control
- Radio / entertainment / infotainment / media system
- Lane assist
- Engine management system
- Auto-park
- Cruise control
- Auto-brake
- Follow-me
- Dashcam

*1 mark each to max 2 for explanation:*

- Limited functions / by example e.g. the system only checks the light and turns lights on/off
- Dedicated microprocessor / by example e.g. there is a microprocessor that is only checking the lights
- Hard to change function / by example e.g. the user cannot make the light system do any other role

**Examiner Guidance:**

- Allow anything that could reasonably be within a car
- If example is unclear (e.g. hazard lights), read explanation for justification — award if justified correctly
- If justification is only generic features of embedded systems (not linked to the example), max 1 mark for explanation
- Do NOT award "built into the car/larger machine" as justification — this is already given in the question

## Virtual Memory

### Q: Explain why the 'Follow Me' system does not need virtual memory [2 marks]

```
keywords: virtual memory, RAM, secondary storage, embedded system, memory, disk thrashing
year: 2024 | paper: J277 | question: 7b(iii) | marks: 2
```

**Question:**  
Explain why the 'Follow Me' system does not need virtual memory.

**Mark Scheme** (1 mark each, max 2):

- Only stores a small amount of data in RAM / only stores specific/few items in RAM
  - → unlikely to run out of RAM / there is enough space in RAM

### Q: Explain how virtual memory can compensate for the lack of RAM [3 marks]

```
keywords: virtual memory, RAM, secondary storage, hard drive, paging, swap, memory management, full RAM
year: 2018 | paper: J276 | question: 4d(i) | marks: 3
```

**Question:**  
The computer will only have 2GB of RAM, but Alicia says that virtual memory can be used instead of adding more RAM.  
Explain how virtual memory can compensate for the lack of RAM in Alicia's computer.

**Mark Scheme** (1 mark per bullet, max 3):

- VM is used when RAM is full
- … part of the secondary storage is used as (temporary) RAM / VM
- Data from RAM is moved to the secondary storage/VM (to make space in RAM)
- RAM can then be filled with new data
- When data in VM is needed it is moved back to RAM

**Examiner Guidance:**

- Many candidates give disadvantages of VM or say the computer can now run more programs — these are NAQ

### Q: Explain why it would be beneficial to get more RAM instead of relying on virtual memory [2 marks]

```
keywords: virtual memory, RAM, disk thrashing, performance, secondary storage, speed, memory access
year: 2018 | paper: J276 | question: 4d(ii) | marks: 2
```

**Question:**  
Explain why it would be beneficial for Alicia to get more RAM instead of relying on virtual memory.

**Mark Scheme** (1 mark per bullet, max 2):

- More RAM will improve the performance of the computer / more RAM will speed up access to data
- Excessive use of VM can cause disk thrashing …
- … which decreases performance
- VM is slower to access than RAM (because it has to go back to RAM first)
- Moving data between RAM and VM takes processor time

**Examiner Guidance:**

- Do NOT award "VM is slower" without quantifying slower at what
