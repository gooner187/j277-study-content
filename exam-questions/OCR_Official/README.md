# J277 OCR Computer Science – Exam Questions Index

This directory contains past exam question banks for the **OCR J277 GCSE Computer Science** specification, organised by topic. Each file contains questions from the SAM (Sample Assessment Material), Practice Papers, and June exam series from 2018–2024.

Questions are sourced from **J277/01** (Computer Systems) and **J277/02** (Computational Thinking, Algorithms and Programming).

---

## How to use this index

- **Finding questions on a topic:** each file is organised by subtopic with keyword tags on every question block, making retrieval fast and precise.
- **Generating a mock paper:** see the mark distribution tables below. Paper 1 is 80 marks; Paper 2 is 80 marks.
- **Cross-referencing:** some questions appear in multiple topic files (clearly flagged). Always prefer the primary file listed below.

### Paper 1 mark distribution guidance (for mock generation)

```
1.1 Systems Architecture       ~10 marks   (10–15% of paper)
1.2 Memory and Storage         ~20 marks   (20–25% of paper)
1.3 Computer Networks          ~12 marks   (12–16% of paper)
1.4 Network Security           ~10 marks   (10–14% of paper)
1.5 Systems Software           ~8 marks    (8–12% of paper)
1.6 Ethical/Legal/Cultural     ~14 marks   (14–18% of paper)
                                ─────────
Total                          ~80 marks
```

### Paper 1 structural rules for mock generation

- Include **at least one diagram question** (star topology or logic diagram from 1.1/1.3)
- Include **at least one conversion calculation** from 1.2 (binary/hex/denary or file size)
- Include **exactly one 8-mark extended response (QER)** from 1.6 — choose from: digital technology in medicine, BYOD, AI on social media, facial recognition, internet access, or smartphone upgrades
- Include **one protocol identification table** from 1.3
- Include **one legislation matching question** from 1.4 or 1.6
- Do **not** duplicate questions from the same year's paper

## Paper 2 – J277/02: Computational Thinking, Algorithms and Programming (80 marks, 1 hour 30 minutes)

| File | Topic code | Topic name | Typical mark range | Question types | Notes |
|---|---|---|---|---|---|

| [2.2-Programming_Fundamentals.md](./2.2-Programming_Fundamentals.md) | 2.2 | Programming Fundamentals | 20–28 marks | Write program, complete program, trace table, identify data types, string manipulation, SQL | Largest topic on Paper 2; data types, variables/constants, arrays (1D/2D), loops, functions, SQL, validation, number representation, sound/image (crossover with 1.2) |

### Paper 2 mark distribution guidance (for mock generation)

```
2.1 Algorithms                          ~16 marks   (18–22% of paper)
2.2 Programming Fundamentals            ~24 marks   (26–32% of paper)
2.3 Producing Robust Programs           ~14 marks   (14–18% of paper)
2.4 Boolean Logic                       ~8 marks    (8–12% of paper)
2.5 Programming Languages and IDEs      ~10 marks   (10–14% of paper)
                                         ─────────
Total                                   ~80 marks
```

### Paper 2 structural rules for mock generation

- Include **at least one algorithm-writing question** (6 marks) from 2.1 or 2.2
- Include **at least one trace table** from 2.1, 2.2, or 2.3
- Include **at least one sorting algorithm** (show stages) from 2.1
- Include **at least one searching algorithm** (binary or linear) from 2.1
- Include **one truth table completion** and **one logic diagram** from 2.4
- Include **one SQL query** from 2.2 or 2.5
- Include **one test plan** from 2.3
- Include **one error identification and correction** (logic or syntax) from 2.3
- Include **one compiler/interpreter or IDE question** from 2.5
- Do **not** duplicate questions from the same year's paper

## Chatbot usage notes

When responding to student queries, use the following guidelines:

- **"What topics are on Paper 1/2?"** → Use the tables above; list topic codes and names with a brief description.
- **"Give me questions about [topic]"** → Open the relevant file and retrieve questions from the matching subtopic section.
- **"Create a mock Paper 1/2"** → Use the mark distribution guidance and structural rules above to select a balanced set of questions across all topic files for that paper.
- **"What's worth the most marks?"** → 1.2 Memory and Storage (Paper 1) and 2.2 Programming Fundamentals (Paper 2) are the highest-weighted topics.
- **"Give me an 8-mark question"** → All 8-mark extended response (QER) questions are in 1.6; all 6-mark algorithm-writing questions are in 2.1, 2.2, or 2.3.
- **"What do examiners penalise?"** → Each question block contains an Examiner Guidance section with specific traps, "Not Enough" points, and "Too Vague" warnings.
