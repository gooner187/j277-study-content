## Syntax Errors – Definition and Examples

### Q: State what is meant by a syntax error and give one example [2 marks]

```
keywords: syntax error, definition, grammar, rules, programming language, does not run, example
year: 2024 | paper: J277/02 | question: 3(a) | marks: 2
```

**Question:**  
State what is meant by the term syntax error. Give one example of a syntax error in a program.

**Mark Scheme:**

**Definition** (1 mark):
- (An error that) breaks the rules/grammar of the programming language
- Stops the program from running // does not allow program to run // crashes the program // does not allow program to translate

**Example** (1 mark, any one):
- Misspelling a keyword (e.g. `printt` instead of `print`)
- Missing / extra symbol (e.g. missing bracket, missing semicolon)
- Mismatched quotes
- Invalid variable or function names (e.g. variable starting with a number or containing a space)
- Incorrect use of operators
- Use of reserved keywords for variables (e.g. `print = 3`)
- Incorrect capitalisation of keywords (e.g. `Print` instead of `print`)
- Incorrect indentation of code blocks
- Missing concatenation (e.g. `print(score x)`)

**Examiner Guidance:**

- BOD code/program etc. for BP1
- "Incorrect grammar" by itself is NE
- Do not allow "stop working", "does not work" — Too Vague
- Do not accept missing quotation marks around a name (could be a variable name)

## Logic Errors – Identify and Correct

### Q: Identify and correct two logic errors in an algorithm [4 marks]

```
keywords: logic errors, identify, correct, line number, total, num1 num2, if statement, range check
year: 2024 | paper: J277/02 | question: 3(b) | marks: 4
```

**Question:**  
The algorithm does not work correctly. Identify the line number of the two logic errors and refine the code to correct each logic error.

```
01  num1 = input("Enter a number")
02  num2 = input("Enter a number")
03  total = num1 + num1
04  if total >= 10 then
05      print("success")
06  else
07      print("warning")
08  endif
```

**Mark Scheme** (1 mark per line number, 1 mark per correction):

- Line **03** – Correction: `total = num1 + num2`
- Line **04** – Correction: `if total >= 10 and total <= 20 then`

**Examiner Guidance:**

- Allow `if 10 <= total <= 20`
- Allow other logically equivalent code (e.g. `total = int(num1) + int(num2)`)
- Allow descriptions of changes as long as clear exactly what will change
- Ignore missing "then" from line 04; ignore capitalisation

### Q: Identify and correct two logic errors in an array-totalling algorithm [4 marks]

```
keywords: logic errors, array, for loop, index, 0-indexed, total, identify and correct, scoreCount
year: 2023 | paper: J277/02 | question: 2(b) | marks: 4
```

**Question:**  
This pseudocode algorithm totals all the numbers in the 0-indexed array `scores`:

```
01  total = 0
02  for scoreCount = 1 to scores.length – 1
03      scores[scoreCount] = total + total
04  next scoreCount
05  print(total)
```

Identify two logic errors and write the refined line to correct each error.

**Mark Scheme** (1 mark per line number, 1 mark per correction):

- Line **02** – Correction: `for scoreCount = 0 to scores.length - 1`
- Line **03** – Correction: `total = scores[scoreCount] + total`  
  (also accept: `total = total + scores[scoreCount]` // `total += scores[scoreCount]`)

**Examiner Guidance:**

- If wrong line number, do not mark correction; if no line number, mark correction only
- Do not penalise if response removes `–1` from `scores.length` as long as it starts at 0
- If both errors fixed on line 03, full marks should be given
- Allow description of change (e.g. "change 1 to 0")

```

**Question:**  
An item is classified as "In demand" if between 5 and 25 inclusive in stock.  
The following program contains syntax and logic errors — refine it:

```
stocklevel = input("Enter stock level")
if stocklevel >= 5 or =< 25 then
    print(Not in demand)
else
    print(In demand)
endif
```

**Mark Scheme** (1 mark each):

- Checking both values (e.g. `or` changed to `and`)
- `if` statement in correct format (checking against `stocklevel` for each condition)
- `if` statement uses correct comparisons (e.g. `>=` and `<=`)
- `print` statements in correct position
- `print` statements include string delimiters around both string outputs

**Example answer:**
```
stocklevel = input("Enter stock level")
if stocklevel >= 5 and stocklevel <= 25 then
    print("In demand")
else
    print("Not in demand")
endif
```

**Examiner Guidance:**

- A candidate who refines the program to work fully but in a different format should gain full marks

## Validation – Writing Algorithms

### Q: Write a validation algorithm for high jump height input [4 marks]

```
keywords: validation, write algorithm, range check, input, output valid not valid, height, centimetres, 40 180
year: 2024 | paper: J277/02 | question: 9(c)(i) | marks: 4
```

**Question:**  
The height jumped (in cm) must be between 40.0 and 180.0 inclusive.  
Write an algorithm to: take the height as input; output "VALID" or "NOT VALID" depending on the height.

**Mark Scheme** (1 mark each):

- BP1: Inputs a value and stores/uses it
- BP2: Checks min value (`>= 40.0` // `< 40`)
- BP3: Checks max value (`<= 180.0` // `> 180`)
- BP4: Outputs both VALID / NOT VALID correctly

**Example answer:**
```
h = input("Enter height jumped")
if h >= 40 and h <= 180 then
    print("valid")
else
    print("not valid")
endif
```

**Examiner Guidance:**

- Answers using AND/OR must be logically correct — do not accept `if height >= 40 and <= 180`
- BP4 needs reasonable attempt at either BP2 or BP3
- Do not accept `=>` (wrong way around); no obvious spaces in variable names

## Validation Methods

### Q: Identify two methods of validation and explain how they apply to an adding game [6 marks]

```
keywords: validation methods, range check, type check, presence check, length check, format check, look up check
year: 2023 | paper: J277/02 | question: 5(b) | marks: 6
```

**Question:**  
Identify two methods of validation and explain how they can be used on a game where players add two random whole numbers between 1 and 10 inclusive.

**Mark Scheme** (1 mark for method name, up to 2 marks for use – max 3 per method):

**Range check:**
- … checks upper/lower boundaries
- … make sure the player's answer is between sensible limits (e.g. 2 to 20 inclusive)
- … by example of program code

**Type check:**
- … making sure the data inputted is of the correct data type
- … make sure answer/input is an integer (or whole number)

**Presence check:**
- … making sure a value is inputted / not blank
- … reference to answer / input
- … by example of program code

**Length check:**
- … limit number of characters // check maximum/minimum string length
- … answer/input must be 1 or 2 characters
- … by example of program code

**Format check:**
- … making sure the data inputted follows a set pattern
- … checking answer/input consists of only 1 or 2 numeric digits
- … by example of program code

**Look up / table check:**
- … making sure data inputted is one from an allowed set of values
- … checking that answer/input is one of [2, 3...20] inclusive
- … by example of program code

**Examiner Guidance:**

- Validation must be applied to the rules of the game; do not accept uses related to inputs not asked for
- For range check, values must be sensible and allow input of 2 to 20 — do not allow 1/10 (answer could exceed)
- Do not accept defensive design elements (e.g. input sanitisation, authentication)

## Testing – Types

### Q: Name and describe one type of test that should be used before releasing a program [2 marks]

```

Complete the table by naming and describing one type of test that should be used on Charlie's program before releasing it.

| Test type | Description |
|---|---|
| | |

**Mark Scheme** (1 mark for name, 1 mark for matching description):

- **Final / terminal testing** — completed at the end of development / before release / to test the product as a whole
- **Iterative / incremental testing** — completed during development / after each module / test module in isolation
- **Normal testing** — test using data that should be accepted // test that is expected to work/pass

- **Black box testing** — testing without access/knowledge of a system's workings
- **White box testing** — testing with access/knowledge of system's workings

**Examiner Guidance:**

- Description must match test type
- Must be a description, not just an example
- Do not accept descriptions that simply repeat the type without further clarification
- Do not accept examples of validation (e.g. type test, range check)

## Writing Algorithms (with robustness focus)

### Q: Write an algorithm for an adding game with scoring [6 marks]

```
keywords: write algorithm, random numbers, score, for loop, if statement, count-controlled loop, adding game
year: 2023 | paper: J277/02 | question: 5(c) | marks: 6
```

**Question:**  
Write an algorithm to play the adding game: the player is asked 3 addition questions; each uses two random whole numbers between 1 and 10 inclusive; if correct, 1 is added to the score; at the end the score is displayed.

**Mark Scheme** (1 mark each):

- BP1: Initialise / declare score (to zero) before use, outside any loop
- BP2: Generates 2 random numbers between 1 and 10
- BP3: Inputs answer from user displaying suitable numbers
- BP4: Checks if input is correct answer …
- BP5: … if correct, adds 1 to score
- BP6: Repeats BP2–5 three times

**Example answer:**
```
score = 0
for count = 1 to 3
    num1 = random(1, 10)
    num2 = random(1, 10)
    ans = input("What is " + num1 + " + " + num2 + "?")
    if ans == num1 + num2 then
        score = score + 1
    endif
next count
print("You scored " + score)
```

**Examiner Guidance:**

- If random numbers chosen, BP3 must use these; if no random numbers, allow manually set values
- BP6 can be awarded for either a loop repeating 3 times or the same code written 3 times
- Do not award BP6 if same numbers used for every question — must pick new values each time
- Do not penalise potential off-by-one errors for looping (Python) or random number generation
