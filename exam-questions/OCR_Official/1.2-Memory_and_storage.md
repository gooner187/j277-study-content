## Hexadecimal Conversion

### Q: Describe how to convert a 2-digit hexadecimal number into denary [3 marks]

```
keywords: hexadecimal to denary, hex conversion, base 16, multiply by 16, via binary, hex digits
year: 2024 | paper: J277 | question: 1d | marks: 3
```

**Question:**  
Describe how to convert a 2-digit hexadecimal number into denary.  
Use an example in your answer.

**Mark Scheme** (1 mark for a correct example; 1 mark each for up to 2 stages described):

**Method 1 – Multiplication:**
- Multiply the left/first digit by 16
- Add the value of the second digit

**Method 2 – Via Binary:**
- Convert each hexadecimal digit into 4-bit binary
- Combine to make an 8-bit binary number
- Convert the 8-bit binary number into denary

**Examiner Guidance:**

- No marks for converting denary to hex
- If the example has an inaccurate result, method marks can still be awarded
- No requirement to show how letter-digits (A–F) are used

### Q: Write a pseudocode algorithm to convert MB to bytes [6 marks]

```
keywords: file size calculation, MB to bytes, kilobytes, algorithm, pseudocode, storage units, conversion
year: 2018 | paper: J276 | question: 1b(ii) | marks: 6
```

**Question:**  
A program needs to calculate the size of files in bytes. The program must:
- Ask the user to input a file size in megabytes
- Calculate and output the number of bytes this represents in a user friendly format  
  (e.g. "There are 5242880 bytes in 5MB")

Write an algorithm using pseudocode to calculate the number of bytes in a given number of megabytes.

**Mark Scheme** (1 mark per bullet, max 6):

- Output asking for file size (in megabytes)
- Taking number of MB as input
- Multiplying by 1024 or 1000
- Multiplying by 1024 or 1000 again (may be same line; must be the final value)
- Outputting the final bytes value …
- … in an appropriate message

**Example solution:**
```
output "Please enter the file size in megabytes"
input numberMB
numberKB = numberMB * 1024
numberBytes = numberKB * 1024
output "There are " & numberBytes & " bytes in " & numberMB & "MB"
```

**Examiner Guidance:**

- Award bullet 5 even if bullets 3 and 4 are wrong — do NOT award if outputting the original input value
- Bullet 4 must be the final calculation — further changes to the bytes value mean bullet 4 cannot be awarded
- `input = value` is incorrect — variable must be on the left
- Input must be stored (e.g. "user input" with no variable = no mark)
- Outputs must have `""` around strings; variable identifiers must NOT have `""`
- Bullets 3 and 4 could instead use ×1,000,000 or ×1,048,576 (award both bullets)

```
keywords: optical storage, solid state storage, discuss, portability, robustness, capacity, cost, 5GB, transfer files, extended response
year: 2019 | paper: J276 | question: 1c(iii) | marks: 8
```

**Question:**  
Kerry was originally going to use an optical storage device to transfer her files.  
Discuss whether an optical or solid state device is the most appropriate media to transfer these files.  
You may want to consider: portability, robustness, capacity, cost.

**Mark Scheme (Levels of Response, max 8 marks):**

**Mark Band 3 – High Level (6–8 marks):**  
Thorough knowledge and understanding across a wide range of considerations. Applied directly and consistently to the context. Weighs up both sides; references impact across all areas. Well-developed, clear, logically structured line of reasoning.

**Mark Band 2 – Mid Level (3–5 marks):**

**Mark Band 1 – Low Level (1–2 marks):**

**0 marks:** No attempt, or not worthy of credit.

**Indicative Content:**

*Portability:*
- Both are small and portable / can easily be moved between locations
- Solid state can be smaller
- Solid state is less likely to break in transit

*Robustness:*
- Optical devices are not robust — easily scratched/damaged while being moved
- Solid state has no moving parts so unlikely to break if dropped

*Capacity:*

## Backup Methods

### Q: Identify the backup method to use and justify [3 marks]

```
keywords: backup, full backup, incremental backup, backup methods, utility software, automatic backup
year: 2018 | paper: J276 | question: 1c(i) | marks: 3
```

**Question:**  
William's computer has utility programs installed including automatic backup.  
William can choose between a full or incremental backup.  
Identify the backup method William should use to backup the computer, justifying your choice.

**Mark Scheme** (either method acceptable; marks for justification, max 3):

**Incremental justifications:**
- Only the changes need to be backed up
- The software/OS/settings are unlikely to have changed between backups
- Small number of files likely to be used/edited between backups
- Takes less time to backup
- Each backup will take less storage space

**Full backup justifications:**
- Backs up all the data/files and software
- Might not take significant time to back up the entire system
- He might only have a small number of files to back up each time
- Safer — more past versions to revert to
- User may have changed settings on the computer
- Faster to restore the backup
- Needs to do a full backup before doing an incremental

**Examiner Guidance:**

- Discussion must match the backup type given
- Allow marks for why the other method is NOT appropriate
