### Q: Rewrite a sale price flowchart as a program [6 marks]


**Question:**  
The following flowchart calculates the price of an item during a sale period:

```
START → Input Price from user →
Is Price greater than 200? Yes → Reduce Price by 40
Is Price greater than 100? Yes → Reduce Price by 20
No (neither condition met) → (no reduction)

```

## Writing Algorithms – Functions and Parameters

### Q: Complete a function moveCharacter() with parameters, selection and return [6 marks]

```
keywords: function, parameters, return, selection, if elseif, pseudocode, write function, moveCharacter
year: 2024 | paper: J277/02 | question: 8(b) | marks: 6
```

**Question:**  
Complete the function `moveCharacter()` that:
- Takes direction (left or right) and current position as parameters
- Subtracts 5 from position if direction is "left"; adds 5 if direction is "right"
- Sets position to 1 if new position < 1; sets position to 512 if new position > 512
- Returns the new position

```
function moveCharacter(direction, position)
    [your answer here]
endfunction
```

**Mark Scheme** (1 mark each):

- BP1: Appropriate use of both parameters; no additional inputs or incorrect overwrites
- BP2: Attempt at selection …
- BP3: … correctly checks direction is "left" and subtracts 5
- BP4: … correctly checks direction is "right" and adds 5
- BP5: Ensures position is between 1 and 512 inclusive
- BP6: Returns the updated position

**Example answer:**
```
function moveCharacter(direction, position)
    if direction == "left" then
        position = position - 5
    elseif direction == "right" then
        position = position + 5
    endif
    if position < 1 then
        position = 1
    elseif position > 512 then
        position = 512
    endif
    return position
endfunction
```

**Examiner Guidance:**

- Allow `else` for BP3/BP4 (validated in preceding question)
- Allow `<=`, `>=` and equivalents (e.g. `<= 0`) for BP5
- Do not award BP5 if placed before BP3 and BP4
- BP6 only given if attempt made at calculating new position
- For minor syntax errors, penalise once then FT

## Sorting Algorithms – Comparison

### Q: Describe one difference between an insertion sort and a bubble sort [2 marks]

```
keywords: insertion sort, bubble sort, difference, comparison, sorted partition, swap, pass, efficiency
year: 2023 | paper: J277/02 | question: 3(c)(i) | marks: 2
```

**Question:**  
Describe one difference between an insertion sort and a bubble sort.

**Mark Scheme** (1 mark each for insertion and bubble sort, max 2):

**Insertion sort:**
- Inserts/moves values into correct position
- Inserts value once (then in correct position)
- Stops when end of array reached // completes in one pass through the array
- Creates a sorted array within an array // has a sorted/unsorted partition
- More efficient/faster than bubble sort … because fewer iterations / comparisons (on average)

**Bubble sort:**
- Compares/swaps pairs of values
- Value is repeatedly moved/swapped until in correct position
- Repeats if a swap has been made // needs multiple passes
- End of array becomes sorted first
- Less efficient/slower than insertion sort … more iterations / comparisons (on average)

**Examiner Guidance:**

- Answer must reference both sorts for 2 marks except when efficiency plus expansion given
- Allow reference to Big O for efficiency
- Do not accept "completes in one iteration" for insertion sort
