## Translators – Compiler vs Interpreter

```
keywords: compiler, interpreter, differences, translate, executable, error reporting, line by line, all at once
year: SAM | paper: J277/02 | question: 5(b) | marks: 4
```

**Question:**  
Either a compiler or an interpreter can translate the code.  
Describe two differences between how a compiler and an interpreter would translate the code.

**Mark Scheme** (1 mark for correct identification, 1 mark for valid description – max 4):

## Translators – Compiler Benefits

### Q: Describe the benefits of using a compiler instead of an interpreter [3 marks]

```
keywords: compiler benefits, advantages over interpreter, executable file, faster execution, no translator needed, source code protection, all errors shown
year: 2024 | paper: J277/02 | question: 7(b) | marks: 3
```

**Question:**  
Describe the benefits of using a compiler instead of an interpreter when writing a program.

**Mark Scheme** (1 mark each, max 3):

- Can produce an executable file
- Program/code runs/executes faster (than interpreted version)
- End users do not need a translator
- Can be run again/multiple times without re-translating // only needs to translate once
- End users have no access to source code // distributed with no source code … cannot steal/copy/modify code
- Shows all/multiple errors // shows errors at the end (of compilation) // creates error file
- Compiler can optimise the code

**Examiner Guidance:**

- Allow in reverse (e.g. "interpreter translates every time")
- Do not allow "no access to source code" unless clearly talking about end users / distribution context
- Do not allow descriptions of how a compiler translates (e.g. "translates whole code in one go")
- "Faster / quicker" by itself is Too Vague

```

```

- BP1: **TeamName**
- BP2: **TblResult**

## SQL – Writing SQL with AND Conditions

### Q: Write an SQL statement to display sensor IDs of door sensors triggered for more than 20 seconds [3 marks]

```
keywords: SQL, SELECT, FROM, WHERE, AND, multiple conditions, SensorID, SensorType, Length, door sensors
year: 2023 | paper: J277/02 | question: 6(d) | marks: 3
```

**Question:**  
Write an SQL statement to display the sensor IDs of the door sensors that have been triggered for more than 20 seconds.

*(Table: events — fields: Date, SensorID, SensorType, Length)*

**Mark Scheme** (1 mark each):

- `SELECT SensorID` (// `SELECT *`)
- `FROM events`
- `WHERE Length > 20 AND SensorType = "Door"` (// reversed order)

**Full correct answer:**
```sql
SELECT SensorID
FROM events
WHERE Length > 20 AND SensorType = "Door"
```

**Examiner Guidance:**

- Max 2 if out of order or anything extra that affects the output
- BP1 can select multiple fields as long as SensorID is included
- Ignore case; only penalise spaces if obvious; field names must be correct
- "Door" must be in quotation marks for BP3
- BP3 can use `==` or `=` for equivalence
- Allow alternative WHERE clauses that are logically correct (e.g. `WHERE length >= 21`)

## File Handling – Procedures

### Q: Write a procedure SaveLogs() that stores data to a text file [6 marks]

```
keywords: file handling, procedure, write file, open file, close file, parameters, writeLine, SaveLogs
year: 2023 | paper: J277/02 | question: 6(e) | marks: 6
```

**Question:**  
Write the procedure `SaveLogs()` that:
- Takes the string of data to be stored as a parameter
- Takes the filename of the text file as a parameter
- Stores the string of data to the text file

**Mark Scheme** (1 mark each):

- BP1: Define procedure SaveLogs …
- BP2: … with two valid parameters
- BP3: Open file (for write/append) …
- BP4: … using the file name passed in as parameter
- BP5: Write data to file …
- BP6: … using the data passed in as parameter
- (BP7): Close file

**Example answer:**
```
procedure SaveLogs(data, filename)
    logFile = open(filename)
    logFile.writeLine(data)
    logFile.close()
endprocedure
```

**Examiner Guidance:**

- Must be clear that answer is a procedure definition; do not credit calling the procedure for BP1
- Allow function definition
- If parameters are later overwritten, do not credit BP2 but FT for BP4 and BP6
- Closing file does not need reference to file name/object — "close file" is enough
- If code given outside of procedure, do not give BP4 and BP6
- Allow FT for multiple occurrences of same mistake (e.g. not using filename correctly for open and close)
