## LAN and WAN

### Q: Identify if a house network is a LAN or WAN and justify [3 marks]

```
keywords: LAN, WAN, local area network, wide area network, identify, justify, own hardware, infrastructure, geographical area
year: 2018 | paper: J276 | question: 2a | marks: 3
```

**Question:**  
A house has computers in each room and a central router. Every room allows both Ethernet and WiFi connections to the router.  
Identify if the house network is a LAN (local area network) or a WAN (wide area network). Justify your choice.

**Mark Scheme:**

- LAN / Local Area Network (1 mark)
- Small distance/geographical area by example (e.g. same building/house) (1 mark)
- Connected by own hardware/infrastructure / not connecting through Internet / no hired/third-party infrastructure / dedicated connection (1 mark)

**Examiner Guidance:**

- Do NOT allow "in a local area" — "local" must be quantified in some way (e.g. same building)
- No marks for WAN

**Question:**

Draw a diagram to show how the five computers, switch and two printers are connected in a star topology.

**Mark Scheme** (1 mark each):

- 5 computers, 2 printers and 1 switch all clearly labelled
- All devices directly connected to the switch (or all computers to switch and each printer to switch/computer/s)
- Only 8 devices total and no additional connections other than to the switch

### Q: Give one benefit and one drawback of star topology over mesh topology [2 marks]

```
keywords: star topology, mesh topology, benefit, drawback, single point of failure, switch, data collisions, add nodes
year: 2024 | paper: J277 | question: 2c(ii) | marks: 2
```

**Question:**  
Give one benefit and one drawback of the office using a star topology instead of a mesh topology.

**Mark Scheme:**

**Benefit (1 mark):**
- Easier to add new nodes / easier to set up
- Central device can monitor/control transmissions
- Faster data transmission
- Fewer data collisions
- If one connection/computer breaks, the network still works
- Less cost of cables

**Drawback (1 mark):**
- If the switch fails, the network fails / single point of failure / reliant on central device
- Extra cost of central device / switch

**Examiner Guidance:**

- "Speed" / "cheaper" etc. on their own are NE
- Read the whole benefit and drawback before awarding marks
- Do NOT award contradictory statements

## IP Addresses – IPv4 and IPv6

### Q: Give one valid example of an IPv4 and one valid example of an IPv6 address [2 marks]

```
keywords: IP address, IPv4, IPv6, internet protocol, network addressing, hexadecimal, denary, format
year: 2024 | paper: J277 | question: 2a(i) | marks: 2
```

**Question:**  
Each computer has an IP address and a MAC address.  
Give one valid example of an IPv4 address and one valid example of an IPv6 address.

**Mark Scheme:**

**IPv4 example:**
- 4 groups of denary numbers between 0 and 255, separated by full stops
- e.g. 123.16.46.72

**IPv6 example:**
- 8 groups of hexadecimal numbers between 0 and FFFF, separated by colons
- A double colon can appear once and replaces any number of groups of consecutive 0000
- e.g. 0252:5985:89ab:cdde:a57f:89ad:efcd:00fe
- e.g. F513:8C:2A::999:0000

**Examiner Guidance:**

- IPv6: each hex number can be between 1 and 4 digits per group

## Network Threats and Preventions

### Q: Describe three threats to a home network and give one prevention for each [9 marks]

```
keywords: network threats, virus, malware, trojan, worm, spyware, phishing, pharming, hacking, brute force, social engineering, ransomware, prevention, firewall, anti-virus, encryption
year: 2018 | paper: J276 | question: 2e | marks: 9
```

**Question:**  
The house owner is concerned about potential threats to the network from being connected to the Internet.  
Describe three possible threats to the computers connected to the network and give one way each threat can be reduced or prevented.

**Mark Scheme** (1 mark for naming threat + 1 mark for description + 1 mark for prevention; max 3 threats; max 9 marks):

**Virus / Trojan / Worm / Malware:**
- A piece of software/code/program that replicates itself / causes damage (e.g. editing/deleting files)
- Prevention: running anti-virus/anti-malware software / don't download from unknown sources / don't click on unknown links

**Spyware / Malware / Keylogger:**
- A piece of software/code that records actions/key presses and sends this data to a third party for analysis
- Prevention: running anti-spyware/anti-malware software / firewall

**Data interception (passive):**
- Data is sent across a network and intercepted by a third party
- Prevention: encryption

**Phishing:**
- An email has a link that when clicked directs the user to a fake website that collects personal data
- Prevention: network policy / firewall

**Pharming:**
- A piece of code installed that redirects the user to a fake website that collects personal data
- Prevention: anti-malware / firewall

**Hacker (unauthorised access):**
- A person attempting to gain unauthorised access to the network/computers/data/files / unauthorised access and then deleting/editing data/files
- Prevention: firewall / strong password / biometrics / penetration testing

**Brute force attack:**
- Person/software using every combination of passwords to gain access
- Prevention: firewall / strong passwords

**Social engineering:**
- Person being the weak point of the system / by example e.g. any example of deception
- Prevention: strong passwords / check validity of sources

**Ransomware:**
- Prevents access to your files unless a ransom is paid
- Prevention: anti-virus / firewall

**Examiner Guidance:**

- Must be relevant to home use — do NOT award: denial of service, SQL injection
- Do NOT allow: adware, spam
- Do NOT allow: backup as a prevention — it does not prevent the threat
- Do NOT allow: encryption for stopping a hacker
- Description must do more than repeat the threat name
- Read whole response; identify the threat first (may not be at the start)
- If no threat identified → no mark for prevention
- Award malware only once: virus and spyware can both get marks, but virus → malware → spyware means spyware is a repeat
