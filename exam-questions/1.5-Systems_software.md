## Operating System Functions – Overview

### Q: Complete a table of OS functions and their tasks [4 marks]

```
keywords: operating system functions, memory management, peripheral management, file management, user interface, OS tasks, complete the table
year: 2024 | paper: J277 | question: 3a | marks: 4
```

**Question:**  
Complete the table by writing the two missing function names and a task performed by the two given functions.

| Function | Task |
|---|---|
| (missing) | Moves data from secondary storage to RAM |
| Peripheral management | (missing) |
| (missing) | Allows the user to create, name and delete folders |
| User interface | (missing) |

**Mark Scheme** (1 mark for function name + 1 mark for matching task):

**Memory management** (BOD: storage management)
- Task given: Moves data from secondary storage to RAM

**Peripheral management**
- Receiving data from input devices
- Transmitting data to output devices
- Installing / downloading device drivers
- Allows communication from input device / to output device

**File management**
- Task given: Allows the user to create, name and delete folders

**User interface**
- Outputting data to the user
- Receiving input from the user
- Allows user to communicate / interact with / control the computer
- Creating / displaying / allowing interaction with a GUI or command prompt interface

**Examiner Guidance:**

- BOD "storage" for "memory" in the first function name
- Peripheral management: allow input and output devices by example
- Do NOT award "folder management" as the function name
- Peripheral management task must extend beyond just "manage" — "manage output devices" is NE

### Q: Identify and describe two functions of an operating system [6 marks]

```
keywords: operating system functions, user interface, memory management, multitasking, peripheral management, drivers, user management, file management, OS describe
year: 2021 | paper: J276 | question: 2a | marks: 6
```

**Question:**  
One type of system software is the operating system.  
Identify and describe two functions of an operating system.

**Mark Scheme** (1 mark for function name + up to 2 marks for description, max 6):

**User Interface:**
- Allows the user to communicate with the hardware
- e.g. GUI / command prompt
- Allows the user to input data / outputs data to the user

**Memory Management / Multitasking:**
- Moves data between RAM and virtual memory
- Gets data from RAM / stores data to RAM
- Switches between processes so fast it appears like multitasking

**Peripheral Management / Drivers:**
- Allows the installation of drivers to communicate with peripherals
- Sends data to/from peripherals
- Sends data to/from buffers

**User Management:**
- Allows the setting up of user accounts
- Security / setting of passwords
- Setting different access rights/levels

**File Management:**
- Creation / editing / renaming of files
- Creation / editing / renaming of folders
- Movement of files/folders

**Examiner Guidance:**

- Read the whole answer for each function
- Max 2 marks for description; 1 mark for function name

## Compression (Utility Software)

*(See also [1.2 Memory and Storage](./1.2-Memory_and_Storage.md#compression--lossless-and-lossy) for compression questions.)*

### Q: Explain how compression software will compress an image file [4 marks]

```
keywords: compression, image compression, algorithm, lossy, lossless, run length encoding, RLE, colour depth, pixels, reduce file size
year: 2019 | paper: J276 | question: 2b(i) | marks: 4
```

**Question:**  
The utility system software provides compression software. Xander uses this to compress an image.  
Explain how the compression software will compress the image file.

**Mark Scheme** (1 mark per bullet, max 4):

- Use an algorithm
- … to remove repeated/unnecessary data
- Could use lossy/lossless
- Lossless will not remove data permanently / lossless means the original file will be restored
- Lossy is permanent deletion / lossy means the original file will not be restored
- Reduce number of pixels / reduce resolution
- Record the changes in the colour for each pixel
- … instead of the colour of every pixel
- Run length encoding (RLE)
- … record the colour and number of consecutive pixels of that colour
- … instead of the colour of every pixel
- Decrease colour depth / decrease number of colours

**Examiner Guidance:**

- "Lossy removes unnecessary data permanently" gets 3 marks: 1 for lossy, 1 for "removes unnecessary data" (if not already awarded), 1 for "permanent"
- Do NOT award "not noticeable to the human eye" or "keeps same/reduces quality" — does not explain how compression works
- Do NOT accept "information" for "data"

## Client-Server Model

### Q: Identify the client in an artist uploading images to a website and justify [3 marks]

```
keywords: client server, client, server, web server, upload, client-server model, identify client, justify
year: 2023 | paper: J277 | question: 5c(i) | marks: 3
```

**Question:**  
The artist uploads images to be displayed on a website. This is a client-server relationship.  
Identify the computer that is acting as the client in this scenario and justify your choice.

**Mark Scheme:**

- Client (1 mark): **Artist's computer / the computer uploading the images** (BOD: "the artist")

- Justification (1 mark each, max 2):
  - Sends the files/data for storage / to the host/web server
  - Performs the user's actions and sends the results to the web server
  - Sends a request to the web server to store/upload its files
  - It does not store data for others to access
  - Confirmation of upload/error is received from server for display

**Examiner Guidance:**

- Incorrect computer → do NOT award justification
- Justification must be about uploading to the web server, not downloading
- Accept "host" for "web server"
- "User's computer" is NE — read justification
- "User viewing the website" is incorrect

---

Justify your choice.

- A smart watch is not a general-purpose computer
- … which means the smart watch has one/limited/specific/dedicated function(s)
