### Von Neumann Architecture

With the very first computers, it was not possible to store programs, and programs were generally input by setting switches. John Von Neumann developed the concept of the stored program computer in the 1940s. The Von Neumann architecture used the idea of holding both programs and data in memory. Data would then move between the memory unit and the processor.

The key components and how they connect:

- **Storage** sits above the Processor (CPU) and communicates with it via a two-way connection
- **Processor (CPU)** sits at the centre — it receives input from Input devices and sends results to Output devices
- **Input devices** feed data into the Processor from the left
- **Output devices** receive data from the Processor on the right
- **Cache** sits below the Processor and communicates with it via a two-way connection
- **Main Memory** sits below Cache and communicates with it via a two-way connection

The Von Neumann model is the foundation of all modern general-purpose computers. Its defining feature is the **stored program concept** — instructions and data are both held in memory and fetched by the CPU as needed.

### Control Unit

The Control Unit coordinates all activities taking place inside the CPU. Its functions:

- Controls the execution of instructions in the correct sequence
- Decodes instructions
- Regulates and controls processor timing using regular pulses from the system clock
- Sends and receives control signals to and from other devices within the computer

---

### The Arithmetic and Logic Unit (ALU)

The ALU carries out three types of operation:

- Logical Operations: AND, OR and NOT
- Shift Operations: bits in a computer word can be shifted left or right by a certain number of places
- Arithmetic Operations: addition, subtraction, multiplication and division

---

### Fetch-Decode-Execute Cycle

When a program is run, it must first be loaded into main memory. The processor then accesses it from memory, running one instruction at a time. This repeating process is called the **Fetch-Decode-Execute cycle** (or simply the Fetch-Execute cycle).

The cycle has three stages:

1. **Fetch** — The Program Counter (PC) holds the address of the next instruction. This address is loaded into the Memory Address Register (MAR). The Control Unit fetches the instruction from that address in main memory and stores it in the Memory Data Register (MDR). The PC then increments to point to the next instruction.
2. **Decode** — The Control Unit (CU) decodes the instruction so it understands what to do.
3. **Execute** — The processor carries out the instruction. Any arithmetic, logic or shift operations are carried out in the Arithmetic Logic Unit (ALU), with results stored in the Accumulator (ACC).

This cycle repeats continuously while a program runs, with one instruction processed at a time.

### Special-Purpose Registers

A register is a special very fast memory location within the CPU used in the execution of instructions.

The four special-purpose registers:

- Memory Address Register (MAR): holds the address (location in memory) of the instruction or piece of data to be fetched or stored
- Memory Data Register (MDR): holds data or a program instruction temporarily when it is fetched from memory or is to be sent to memory
- Program Counter (PC): holds the memory address of the next instruction to be processed
- Accumulator (ACC): a special-purpose memory location where results of operations carried out in the ALU are temporarily stored. Most processors have several general-purpose registers instead of a single accumulator, used as temporary fast storage.

Example — assembly code that fetches a value, adds 80 to it, and stores the result:

```
LDA 2560    loads the data at address 2560 into the accumulator (ACC)
ADD #80     adds the value 80 to the present content of the accumulator
STA 2560    stores the content of the accumulator in memory location 2560
```

To execute LDA 2560: the address 2560 is passed to the MAR → the computer transfers the data from that memory location to the MDR → the data passes from the MDR to the ACC for use in the calculation.

When you write a = a + 80 in Python or Visual Basic, it is broken down into these three basic instructions.

### CPU Performance Factors

The three main factors affecting CPU performance are:

- Clock speed
- Cache size
- Number of cores

---

### Clock Speed

Clock speed is the speed at which a processor operates, measured in hertz (Hz) — the number of cycles per second. Everything the processor does occurs on the 'tick' of the clock.

- 1 Hz = one clock cycle per second
- Clock speed is measured in Gigahertz (GHz) — one billion cycles per second
- Typical PC clock speeds: 2 GHz to 4 GHz
- The greater the clock speed, the faster instructions will be executed

A 4 GHz processor should in theory operate twice as fast as a 2 GHz processor, but other components can create bottlenecks that slow the whole system down — like a three-lane motorway feeding into a single-lane road into town.

---

### Cache Memory

A bottleneck can occur because reading from and writing to RAM (main memory) is much slower than the speed at which the CPU can work. Faster memory technologies exist but are more expensive.

Cache is a small amount of very fast (but more costly) memory where frequently used instructions or data are stored temporarily. Cache acts as an intermediary between RAM and the CPU:

- Data frequently used by the CPU is available much more quickly from cache
- The processor accesses main memory less often → CPU works faster → performance increases
- If the required data is not in cache, it must be fetched from RAM

Typical sizes (2020): 8–16 GB of RAM vs 2 MB of cache. There are 1000 MB in 1 GB, so a typical computer has thousands of times more RAM than cache.

The more cache a computer has, the more data and instructions can be held close to the CPU, improving performance.

---

### Number of Cores

A core is an individual processing unit within a CPU. Modern CPUs typically include more than one core:

- Dual-core: two processing units
- Quad-core: four processing units

Multiple cores allow parallel processing — two or four instructions can be executed simultaneously.

In theory, two cores means twice as many operations per second, but this is not always the case:

- A program is a series of instructions that must be done in order
- Multiple cores can work on different programs running in parallel
- Unless the computer is designed to exploit multiple cores, a quad-core processor is not necessarily four times faster
- However, a multi-core PC executing many tasks simultaneously will operate faster than a single-core processor

### Embedded Systems

An embedded system is a small computer built into a piece of equipment, designed to perform a specific function.

Examples: vehicles, cameras, medical equipment, aircraft, vending machines, ovens, fridges, mobile phones, satellite navigation devices, televisions, digital clocks, lifts.

---

### Characteristics of an Embedded System

- Reliability and use of minimal resources (ROM, timers, sensors, actuators)
- The control program is held in ROM and cannot be changed. It is written in a machine-efficient language (minimises memory use, maximises speed). Assembly language may be used for critical parts.
- Very limited operating system — unlike a desktop OS, it does not need to be loaded at startup as it is stored in ROM. It can only run a single application.
- Very simple user interface (e.g. buttons on a microwave or washing machine). Some embedded systems have no user interface at all.
- Some have sensors to measure external stimuli and react accordingly — e.g. a central heating system uses buttons and sliders to control timing and temperature.

### Misconceptions

- misconception: The MAR holds the data being fetched
  reality: The MAR holds the ADDRESS of the data or instruction to be fetched. The MDR holds the actual DATA. They are two separate registers with distinct roles.
  trigger: When students confuse what MAR and MDR store
  correction_prompt: MAR = Memory ADDRESS Register — it stores where to look. MDR = Memory DATA Register — it stores what was found. Think: Address first (MAR), then Data arrives (MDR).

- misconception: A faster clock speed always means a faster computer
  reality: Clock speed is one of three performance factors. Other components (RAM speed, cache size, bus speed) can create bottlenecks. A faster clock speed helps but cannot overcome a slow component elsewhere in the system.
  trigger: When students say doubling the clock speed will double the computer's speed
  correction_prompt: Clock speed determines how fast the CPU can execute instructions, but the whole system must keep up. A slow RAM or small cache can bottleneck performance just like a single-lane road limits a motorway.

- misconception: Cache is just another name for RAM
  reality: Cache and RAM are different types of memory. Cache is much smaller, much faster, and much more expensive per byte. RAM is larger, slower, and cheaper. Cache sits between the CPU and RAM to reduce the number of slow RAM accesses.
  trigger: When students use cache and RAM interchangeably
  correction_prompt: Cache is NOT RAM. Cache is a small, extremely fast memory buffer between the CPU and RAM. RAM is the main memory where programs and data are stored. Cache holds only the most frequently accessed data.

- misconception: A quad-core processor is always four times faster than a single-core processor
  reality: Multiple cores allow parallel processing but most programs are sequential — they must execute steps in order. Quad-core is most beneficial when running multiple programs simultaneously, not necessarily four times faster for a single task.
  trigger: When students assume core count directly multiplies performance
  correction_prompt: More cores help when running multiple programs at the same time. A single sequential program cannot always use more than one core. Quad-core is not guaranteed to be 4× faster — it depends on how well the software is designed to use multiple cores.
