### 8.4 – Translators and Facilities of Languages

### Classification of Programming Languages

Programming languages fall into two categories:

- High-level languages: Python, Visual Basic, Java, C++, Delphi — look like English or maths, independent of hardware
- Low-level languages: assembly language and machine code — processor-specific

High-level languages were invented because humans find it easier to write programs that look like natural language and are suited to the type of problem being solved. Different languages suit different problems:

- Visual Basic: forms-based data processing
- Java: smartphone applications
- SQL: searching and maintaining databases
- HTML, CSS, JavaScript: creating web pages and websites

Machine code is the binary code that runs directly on the processor. Assembly language uses mnemonics — abbreviated text commands that map directly to machine code instructions:

- LDA (LOAD)
- STO (STORE)
- ADD

There is one assembly language for each processor architecture. Machine code is also processor-specific.

### Characteristics and Use of Programming Languages

High-level languages:

- Easier to learn and faster to program in
- Statements look like English or maths — easier to read, debug and maintain
- Specialised high-level languages exist for specific applications (SQL for databases, HTML/CSS/JS for web)

Assembly language:

- Often used in embedded systems — computer systems that control washing machines, automobiles, traffic lights
- Suitable because:
  - Gives the programmer complete control over specific hardware components
  - Very efficient code can be written for a particular processor — occupies less memory and executes faster than compiled high-level language

---

### Translating Programs into Machine Code

Whatever language a program is written in, it must be translated into machine code to run on the processor. Translators are a type of system software. There are three types:

Assembler:

- Converts assembly language into machine code
- Simple conversion — every assembly language instruction translates into a single machine code instruction

Compiler:

- Translates a high-level language program into machine code
- More complex — a single instruction can result in many machine code instructions
- Different compilers are needed for different processor types
- The programmer's code is called source code
- The machine code produced is called object code
- Commercial software is sold as object code — the customer does not need the compiler to run it and cannot see or copy the source code

Interpreter:

- Also translates high-level language into machine code
- Translates and executes one line at a time — no object code is produced
- The interpreter must be installed on the computer to run the software
- Useful during program development — easy to pinpoint errors as the program stops and highlights the syntax error at the point it occurs
- Slower than compiled code — a loop performed 10,000 times means the lines inside are translated 10,000 times

Note: Java compiles to an intermediate bytecode which can then be interpreted and run on many different processor types using a bytecode interpreter — this makes Java apps portable across different smartphones.

---

### Compiler vs Interpreter — Comparison

```
Compiler                                     | Interpreter
Translates whole program → object code       | Translates and executes one line at a time
Compiled program executes faster             | Interpreted programs execute more slowly
Customer does not need compiler to run       | Customer must have interpreter installed
Customer cannot see source code              | Customer can see and potentially copy source code
Used for finished software sold to users     | Used during development and for bytecode
```

### 8.5 – The Integrated Development Environment

An IDE (Integrated Development Environment) is a tool provided with high-level languages (e.g. Python, Visual Basic, C#) that gives the programmer all the facilities needed to write and run programs.

Four core facilities:

- Enter, edit and save source code
- Identify and correct syntax errors
- Translate source code into machine code
- Execute (run) the program

Python has two modes:

- Interactive mode: instructions are executed as soon as Enter is pressed — useful for trying out single instructions but programs cannot be saved
- Script mode: used to write, save and run full programs

---

### The Editor

The editor is the component of the IDE used to enter and edit program code. Features include:

- Enter a new program, save it, and edit an existing program
- Find and replace all occurrences of text
- Auto-indent statements inside selection and iteration constructs (if...then...else, while...endwhile)
- Auto-complete statements
- Colour-code keywords (e.g. if, then, else), comments, text in quotes, etc.
- Pretty printing
- Break points to stop the program running at specific points
- Watch windows to monitor changes in variable values during execution

---

### Error Diagnostics

Once a program has been entered, the IDE translates it using the interpreter or compiler. The IDE runs error diagnostics on each line. If syntax errors are found, the programmer can:

- Correct the syntax errors
- Save and retranslate
- Run again

After all syntax errors are corrected, the IDE may discover and report a runtime error — an error that only occurs when the program is actually executing. Example:

```
line 10, in <module>
    numScores = numScores + 1
NameError: name 'numScores' is not defined
```

---

### Run-Time Environment

After all errors are corrected, the program runs in a separate window that:

- Displays prompts for the user to enter values
- Accepts user input
- Displays results

Example run of a test scores program:

```
Enter next score: 4
Enter next score: 8
Enter next score: 6
Enter next score: 5
Enter next score: 5
Enter next score: 7
Enter next score: 3
Enter next score: 2
Enter next score: -1
Average score: 5.0
>>>
```

### Misconceptions

- misconception: Compilers and interpreters do the same job
  reality: Both translate high-level code to machine code, but they work differently. A compiler translates the whole program once and produces object code. An interpreter translates and executes one line at a time and produces no object code.
  trigger: When students say "a compiler is just a type of interpreter" or treat them as interchangeable
  correction_prompt: Compiler = translate ALL at once → save as object code → run fast, no translator needed later. Interpreter = translate ONE LINE at a time → run slowly, translator always needed. Different tools for different purposes.

- misconception: The customer gets the source code when they buy software
  reality: Commercial software is sold as object code (compiled machine code). The customer never sees the source code and does not need the compiler. This protects the developer's intellectual property.
  trigger: When students assume buyers can modify or read commercial software
  correction_prompt: When you buy software, you get the compiled object code — binary that runs directly on your processor. The source code stays with the developer. That is why you cannot legally modify or redistribute most commercial software.

- misconception: Interpreted programs are always slower because interpreters are less capable
  reality: Interpreted programs are slower because the translation happens at runtime — every line is translated every time it is executed. A loop run 10,000 times means translating those lines 10,000 times. The interpreter itself is not less capable.
  trigger: When students say interpreters are "worse" than compilers
  correction_prompt: Interpreters are slower for execution but better for development — they stop at the exact line with an error, making debugging easier. Compilers are better for finished software that runs frequently.

- misconception: An IDE is just a text editor
  reality: An IDE is a complete development environment. As well as editing code, it provides syntax highlighting, auto-indent, auto-complete, error diagnostics, a translator, a run-time environment, breakpoints, and watch windows.
  trigger: When students describe an IDE only as a place to type code
  correction_prompt: A simple text editor only lets you type. An IDE actively helps you write, test and debug code — it highlights errors, completes statements, lets you step through code line by line, and runs the program in a managed environment.
