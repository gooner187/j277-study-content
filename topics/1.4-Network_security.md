### Malware

**Malware** is malicious software installed without the user's knowledge. Security hazards include malware, hackers, blagging, phishing, pharming, hardware and software faults, and other network users.

**Virus:**

- Embeds itself in other programs, infecting them
- Installed without the user's knowledge
- Can spread to other users and computers if an infected file is sent to someone else

**Worm:**

- A standalone program — does not need to attach itself to an existing program to spread
- May scan the Internet automatically looking for other computers to infect

**Trojan:**

- Masquerades as having one legitimate purpose but actually has another
- Named after the Ancient Greek story of the Trojan Horse
- Normally spread by email — the user is invited to click a link for a routine or interesting purpose
- Executing the link gives the controller unauthorised access to the computer

---

### Social Engineering

**Social engineering** means tricking or persuading individuals into revealing confidential or personal information (such as passwords or bank details) which can then be used to gain unauthorised access to computer systems, bank accounts, or other confidential information.

**Phishing:**

- Emails designed to steal money, obtain login details, or steal an identity
- The user receives an email inviting them to click a link (e.g. to download a free game or verify bank details)
- The criminal then asks the user to enter sensitive data: bank account numbers, passwords, name, address, date of birth, etc.

**Blagging:**

- Knowingly or recklessly obtaining or disclosing personal data without the consent of the data owner
- Example: a dishonest employee persuades a colleague to reveal their password by pretending to need it to install software
- This gives access to computer files and can lead to security breaches or identity theft
- Can also occur on social networking sites where people are tricked into revealing personal data
- Prevention: provide **security training** to employees so they do not fall for these tricks

### Human Error

People are often the **weakest point** in security systems. Security hazards arising from human error include:

- Employees plugging in unknown storage devices (memory sticks, USB drives) without considering security risks
  - Case study: researchers scattered memory sticks in company car parks — nearly 60% of workers who found them used them without considering security risks; this rose to 90% when an official logo was printed on the device
- Losing devices containing sensitive data (e.g. in 2010, GCHQ lost 35 laptops containing potentially sensitive data)
- Password reuse — Internet users typically manage ~25 devices/applications but only use 5 or 6 unique passwords across all of them

Managers must be aware of these shortcomings and provide adequate security awareness and training.

### Hacking — Brute Force Attacks

**Hacking** involves gaining unauthorised access to confidential information.

**Brute force attack** — a method of discovering a password by systematically trying combinations:

- **Simple brute force:** uses a dictionary of commonly used passwords, cycling through them until access is gained
- **Complex brute force:** tries every possible combination of letters, numbers, and symbols found on a typical keyboard until the correct password is found

---

### Denial of Service (DoS) Attacks

A **Denial of Service (DoS) attack** attempts to prevent legitimate users from accessing a website or service.

**Methods:**

- Flooding a company's network with useless traffic until it becomes overloaded and crashes or appears unresponsive
- Sending millions of spam messages to an email account until the ISP's storage limit is reached, preventing legitimate emails from arriving

**Mitigations:**

- Virus checkers
- Firewalls (which restrict traffic entering a computer)
- Email filters

---

### Data Interception and Theft

**Shouldering** — physically looking over someone's shoulder while they type a PIN or password.

Other physical theft methods:

- Gaining access to a computer left logged in while the user is away

**Prevention measures:**

- Destroy paper documents when no longer needed, or file in a secure location
- Log off or lock the computer when stepping away from the desk
- Lock the room containing the computer as an additional precaution

**Man-in-the-middle attack** — an attacker secretly intercepts communications between two parties and, in some cases, modifies the communications between them (similar to a wiretap).

### SQL Injection

**SQL (Structured Query Language) injection** is a technique that exploits security weaknesses in a website by inserting malicious code into a database field.

- Can be used to corrupt or disable websites
- Can spread viruses and malware
- Can steal sensitive data: credit card numbers, National Insurance numbers, passwords, personal data
- One of the most dangerous and well-known security vulnerabilities on the Internet

**How SQL injection works — example:**

A database login query is structured as:

```
SELECT *
FROM usersList
WHERE (Username = $username)
AND (Password = $password);
```

A hacker inputs `$username) OR (1=1` as the username and `$password) OR (1=1` as the password. The query becomes:

```
SELECT *
FROM usersList
WHERE (Username = $username) OR (1=1)
AND (Password = $password) OR (1=1);
```

Since `1=1` is **always true**, the condition is satisfied for every record — giving the hacker access to the entire database.

**Protection against SQL injection:**

- Check the parameters used in the SQL query to ensure they follow a strict, expected pattern (input validation)
- Limit permissions to use the database by setting **access levels**

### Misconceptions

- misconception: A virus and a worm are the same thing
  reality: Both are malware, but they spread differently. A virus embeds itself in existing programs and spreads when an infected file is shared. A worm is standalone — it does not attach to other programs and can self-propagate automatically across networks without user interaction.
  trigger: When students use virus and worm interchangeably
  correction_prompt: Viruses and worms are both malware but spread in different ways. A virus needs to attach itself to an existing program — it spreads when someone shares an infected file. A worm is standalone and does not need a host program — it can scan the network and spread automatically without any user action. The key distinction is whether it needs to infect another file to spread.

- misconception: Phishing only happens via email
  reality: The textbook defines phishing in the context of email, but phishing attacks also occur via fake websites, SMS messages (smishing), and phone calls (vishing). For the OCR exam, the email-based definition is sufficient, but students should understand that social engineering takes many forms.
  trigger: When students say phishing is exclusively an email-based attack
  correction_prompt: In the OCR textbook, phishing is described as email-based — and that is the definition you need for the exam. In the real world, phishing also happens via fake websites, text messages, and phone calls. For exam purposes, the key features are: it tricks the user into revealing personal data, and it usually involves a link or request that appears legitimate.

- misconception: SQL injection only affects the login page of a website
  reality: SQL injection can exploit any database-connected input field on a website, not just login forms. Any field that passes user input directly into a SQL query without validation is potentially vulnerable — search boxes, contact forms, order forms, etc.
  trigger: When students say SQL injection only works on login systems
  correction_prompt: SQL injection can target any part of a website that passes user input into a database query — not just login pages. Search fields, order forms, and any other input that interacts with a database are potential targets. The principle is the same: if user input is not properly validated, a hacker can inject SQL code to manipulate the database query.

### Penetration Testing

Vulnerability of a computer network is often due to a flawed system which is open to attack. **Penetration testing** is used to find security weaknesses before attackers can exploit them.

**The five steps of penetration testing:**

1. Gather information about the target of possible attacks
2. Identify possible entry points
3. Attempt to break in
4. Report back the findings and fix them
5. Repeat

**External penetration test:**

- Targets email servers, web servers, or firewalls
- Objective: determine whether a hacker can get in, and if so, how far they can go and what they can do

**Internal penetration test:**

- Puts the tester in the position of an employee with standard access rights
- Objective: determine how much damage a disgruntled or dishonest employee could cause

### Anti-Malware Software

**Anti-malware software** protects a computer in three ways:

- **Prevents installation** — stops harmful programs from being installed on the computer
- **Prevents modification** — prevents important files (such as the operating system) from being changed or deleted
- **Detects and removes** — if a virus manages to install itself, the software detects it during regular scans and removes it

Anti-malware software must be **kept up to date** — new viruses are created regularly, so the software requires regular updates from the Internet to remain effective.

### Firewall

A **firewall** is designed to prevent unauthorised access to or from a private network or intranet.

- All messages entering or leaving the intranet pass through the firewall
- The firewall examines each message and blocks those that do not meet specified security criteria

**Firewall criteria may include:**

- Where the access is from (the computer's address)
- The type of traffic (e.g. .exe files which may carry viruses)
- Specific website addresses

**What a firewall can do:**

- Block inbound unauthorised access from the Internet
- Prevent computers on the network from accessing specific sites or categories of site (e.g. social networking during work hours, cricket during a school day)
- Act as a proxy server to stop certain traffic leaving the local network onto the Internet

**Firewall availability:**

- Operating systems such as Windows include built-in firewall utilities
- Firewall software can also be purchased or downloaded separately
- Some banks provide free firewall software to customers using Internet banking

### Password Protection

The easiest way to stop unauthorised access is to use a **combination of username and password**.

**Characteristics of a strong password:**

- Not easy to guess
- Contains random letters, numbers, and symbols
- At least eight characters long
- Not shared with others or written down (e.g. on a Post-it note under the keyboard)

**Additional password security measures:**

- Account lockout after a certain number of failed login attempts (protects against brute force attacks)
- Using validation techniques and verification — checking format and field length when a username and password are entered
- Setting **access levels** to minimise risk even if database security is compromised

**Note on password rotation policies:** making employees change passwords monthly is less effective than it appears — users tend to make predictable changes (e.g. adding the month number) that are easy to guess.

---

### User Access Levels

**User access levels** restrict what individual users can read, modify, or delete on a network.

- Users can only access the files, folders, or systems they need for their role
- Access types include: Read-Only, Read-Write, Full Control, Modify

**Examples:**

- A student at school can read files in a shared area but not edit them (Read-Only); a teacher has Read-Write access
- In a workplace, the Accounts staff can access payroll details; other departments cannot
- The Data Protection Act requires employers to keep personal data secure — setting appropriate access rights is a legal responsibility

---

### Encryption

Network transmissions can be intercepted by devices that scan cable emissions — no physical connection to the network is needed. Intercepted data can be read without leaving any trace.

**Encryption** protects against this by changing data before transmission so it can only be deciphered by someone with the appropriate key. To anyone intercepting the message, it appears unintelligible.

**Example in practice:**

- When accessing Internet banking or making an online purchase, the address bar changes from HTTP to HTTPS
- HTTPS works like HTTP but the data is encrypted — payment details and personal information are kept secure

---

### Physical Security

Physical security protects against unauthorised physical access to premises and hardware.

**Physical security measures:**

- **Perimeter barriers** — security barriers at the entrance that can only be opened by a guard or by entering a PIN or ID
- **CCTV cameras** — used to detect intruders both inside and outside the building
- **Security locks** — entrances that can only be unlocked by authorised personnel; receptionists control visitor access
- **Internal barriers** — barriers between the reception area and the rest of the building to prevent unauthorised access beyond the entrance

### Misconceptions

- misconception: A firewall only blocks incoming threats from the Internet
  reality: A firewall controls both inbound and outbound traffic. It can block external attackers from getting in, but it can also prevent internal users from accessing certain websites or sending certain types of data out. The firewall acts as a gatekeeper for all traffic crossing the network boundary in either direction.
  trigger: When students say firewalls only stop incoming attacks
  correction_prompt: A firewall works in both directions. It blocks unwanted traffic coming into the network — like hackers and malicious files. But it can also block traffic going out — for example, preventing staff from accessing social networking sites during work hours, or stopping students from reaching games websites at school. The firewall examines every message entering or leaving the network.

- misconception: Encryption prevents data from being intercepted
  reality: Encryption does not prevent interception — data can still be captured. What encryption does is make the captured data unreadable. Without the correct key, intercepted data appears as unintelligible characters. The protection is in the unreadability, not the prevention of capture.
  trigger: When students say encryption stops hackers from seeing data
  correction_prompt: Encryption does not stop data from being intercepted — it makes the intercepted data unreadable. A hacker could still capture encrypted network traffic, but without the correct decryption key, it would appear as meaningless characters. The protection encryption provides is confidentiality, not invisibility.

- misconception: Penetration testing is the same as hacking
  reality: Penetration testing is authorised, structured, and carried out to identify and report vulnerabilities so they can be fixed. Hacking is unauthorised access with the intent to exploit or damage. Penetration testers are employed by organisations specifically to find weaknesses before malicious hackers do.
  trigger: When students describe penetration testing as a form of hacking
  correction_prompt: Penetration testing and hacking use similar techniques, but they are very different in purpose and legality. Penetration testing is authorised — the organisation employs a tester to find and report weaknesses. The goal is to fix vulnerabilities before attackers find them. Hacking is unauthorised and typically intends to steal, damage, or exploit. One is a security service; the other is a crime.
