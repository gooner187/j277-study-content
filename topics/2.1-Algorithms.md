### Computational Thinking

Computer Science is about studying problems and figuring out how to solve them. Problems can be:

- Mathematical (e.g. finding all prime numbers less than a million)
- Less well-defined (e.g. detecting when a platform is dangerously crowded, or when a computer has a virus)

Some of the key concepts in computational thinking include:

- abstraction
- decomposition
- algorithmic thinking

---

### Abstraction

Abstraction involves removing unnecessary details from a problem in order to solve it.

Example — recognising a crowded platform:

- The computer can ignore: background lights, clothing colours, whether people carry rucksacks, gender
- The only important fact is: how many heads can be detected

Abstraction separates the 'logical' from the 'physical':

- The London Underground map shows only which stations are on which line and the best route from A to B
- It ignores exact distances between stations and the direction of travel at any moment

Everyday examples of abstraction:

- Using a computer without knowing how it works internally
- Driving a car without understanding the engine mechanics
- A driver, a child in the back seat, and a mechanic all have very different views of a car — we abstract away what we don't need

---

### Abstraction in Simulation

Abstraction is used in thousands of ways to aid problem-solving. One common method is simulation — building a model of a problem and finding out what happens under different circumstances.

A simulation is more likely to be a 'logical' model — one that describes basic facts and performs calculations to predict what will happen.

Examples of simulations:

- Climate change models
- Financial models (e.g. a 'lemonade stand model' to calculate whether a business will make a profit)
- Population models to predict the likely population in 20 years' time based on current trends
- Queueing models to estimate how many toll-booths will be needed on a new motorway, or how many checkouts are needed in a supermarket

---

### Decomposition

Decomposition involves breaking down a problem into smaller, simpler steps or stages.

A structure diagram can be used to show the structure of a problem, its subsections, and links to other subsections.

Example — writing a computer game:

- Computer game breaks down into: Instructions, Levels, Sounds and animation
- Instructions breaks down into: Joystick controls, Keyboard controls
- Levels breaks down into: Level 1 of game, Level 2 of game, Level 3 of game
- Sounds and animation breaks down into: Background music, Character movement

Each box can be further sub-divided until each box represents a single, simple sub-task. When tasks are decomposed in this way, it becomes much easier to solve the problem. Each small part is a manageable subproblem for which the steps in a solution can be written down.

---

### Algorithmic Thinking

An algorithm is a series of steps a program must perform to solve a problem.

A working algorithm will always:

- Finish and return an answer
- Perform the task it was supposed to

When writing algorithms, it is important to identify the input, processing and output:

- input: one or more values entered by a user, a reading from a sensor or device, or a value coded within the program
- processing: defines the steps to solve the problem or actions to take when a particular event occurs
- output: the answer to a problem, or the result of actions initiated by the program

Example algorithm (finds the highest mark from 10 entered marks):

```
x = 0
repeat 10 times
    ask user to enter a mark
    accept the mark
    if mark > x then x = mark
endrepeat
print x
```

- input: the values entered by the user
- output: the value of x after processing has taken place

### 6.2 – Searching Algorithms

Before writing algorithms, it is important to understand well-known algorithms for searching and sorting — both very common operations in the real world.

Thousands of software applications (e.g. databases, commercial search engines like Google) depend on the ability to search through huge amounts of data to find a particular item.

---

### Linear Search

A linear search is used when data is unsorted. It starts at the beginning and looks at every item until the required item is found.

- Best case: item is near the beginning of the list
- Worst case: item is at the end of the list (or not in the list at all)

For a list of 10,000 unsorted names:

- On average, 5,000 items will need to be examined
- If the item is not in the list, all 10,000 items will be examined

Algorithm for a linear search:

```
1.  found = False
2.  Start at the first name
3.  do
4.      Examine the current name in the list
5.      if it's the one you are looking for then found = True
6.  until found = True OR reach end of list
7.  if found = True then
8.      print(name)
9.  else
10.     print("Not found")
11. endif
```

Example — searching for 5 in: 14, 2, 3, 11, 1, 9, 5, 8, 10, 6

- Items examined: 14, 2, 3, 11, 1, 9, 5 (found on 7th comparison)

---

### Binary Search

A binary search can only be used when the list is sorted (numerical or alphabetical order). It is more efficient than a linear search.

It works by repeatedly dividing in half the portion of the list that could contain the required item, until there is only one item left.

Algorithm for a binary search:

```
1.  found = False
2.  while there are more items in the list AND found == False
3.      Examine the middle data item in the list
4.      if this is the required item then
5.          found = True
6.      else
7.          if required item > middle item then
8.              discard the first half of the list
9.          else
10.             discard the second half of the list
11.         endif
12.     endif
13. endwhile
```

Example — searching for 50 in: 15, 21, 29, 32, 37, 40, 42, 43, 48, 50, 60, 64, 77, 81, 90

- Stage 1: middle term is 43 — discard all items less than or equal to 43 → remaining: 48, 50, 60, 64, 77, 81, 90
- Stage 2: middle term is 64 — discard all items greater than or equal to 64 → remaining: 48, 50, 60
- Stage 3: middle term is 50 — item found

Note: if there are an even number of items, the fourth (not the fifth) item of 8 is taken as the middle item.

### Misconceptions

- misconception: Abstraction means making something vague or unclear
  reality: Abstraction means removing irrelevant detail to focus on what matters. It makes problems clearer and easier to solve, not harder.
  trigger: When students describe abstraction as 'ignoring everything' or express confusion about what it means
  correction_prompt: A common misunderstanding is that abstraction removes useful information. In fact, abstraction keeps only the relevant details — for example, a tube map removes distance but keeps the connections that matter for navigation.

- misconception: Decomposition means just listing the steps of a solution
  reality: Decomposition is the process of breaking a complex problem into smaller sub-problems, each of which can be solved independently. It is about structure, not just sequencing.
  trigger: When students confuse decomposition with writing an algorithm
  correction_prompt: Decomposition is about identifying the parts of a problem, not writing the solution steps. Think of it as drawing the structure diagram before writing any code.

- misconception: Computational thinking only applies to computers
  reality: Computational thinking — abstraction, decomposition, and algorithmic thinking — can be applied to any complex problem, not just those solved by computers.
  trigger: When students assume these concepts are only relevant to programming
  correction_prompt: Computational thinking is a general problem-solving approach. Architects, scientists, and managers all use decomposition and abstraction — computers just make it more formal.

### 6.3 – Sorting Algorithms

The binary search method required data to be sorted before the search could take place. There are many sorting algorithms; three are covered here:

- Bubble sort
- Insertion sort
- Merge sort

---

### Bubble Sort

A bubble sort works by repeatedly going through the list, comparing each pair of adjacent elements. If the elements are in the wrong order they are swapped.

Swap algorithm (uses a temporary variable):

```
temp = a
a = b
b = temp
```

How bubble sort works:

- On each pass, the largest unsorted item 'bubbles' to the end of the list
- After pass 1, the largest item is in the correct position
- After pass 2, the two largest items are in the correct position
- Each subsequent pass checks one fewer item

Example — sorting: 9, 5, 4, 15, 3, 8, 11, 2

- Pass 1 end: 5, 4, 9, 3, 8, 11, 2, 15
- Pass 2 end: 4, 5, 3, 8, 9, 2, 11, 15
- Pass 3 end: 4, 3, 5, 8, 2, 9, 11, 15
- Pass 4 end: 3, 4, 5, 2, 8, 9, 11, 15
- Pass 5 end: 3, 4, 2, 5, 8, 9, 11, 15
- Pass 6 end: 3, 2, 4, 5, 8, 9, 11, 15
- Pass 7 end: 2, 3, 4, 5, 8, 9, 11, 15

Optimisation — using a swapMade flag:

- If no swaps are made during a pass, the list is already sorted and the algorithm can stop early
- This avoids unnecessary passes on a partially sorted list

Full bubble sort algorithm:

```
//Bubble sort
numbers = [9, 5, 4, 15, 3, 8, 11, 2]
numItems = len(numbers)
passNumber = len(numbers) - 1
swapMade = True
while passNumber > 0 AND swapMade == True
    swapMade = False
    for j = 0 to passNumber - 1
        if numbers[j] > numbers[j + 1]
            temp = numbers[j]
            numbers[j] = numbers[j + 1]
            numbers[j + 1] = temp
            swapMade = True
        endif
    next j
    passNumber = passNumber - 1
    print(numbers)
endwhile
```

### Insertion Sort

An insertion sort takes one data item at a time from the unsorted list and places it in the correct location in the sorted portion of the list. This is similar to how you might sort a hand of cards.

- More efficient than bubble sort
- Less efficient than merge sort or quick sort

Algorithm:

```
//Insertion sort
alist = [9, 5, 4, 15, 3, 8, 11, 2]
listLength = len(alist)
for index = 1 to listLength - 1
    current = alist[index]
    pos = index
    while pos > 0 AND alist[pos - 1] > current
        alist[pos] = alist[pos - 1]
        pos = pos - 1
    endwhile
    alist[pos] = current
next index
print("Sorted list: ", alist)
```

Example — sorting: 9, 5, 4, 15, 3, 8, 11, 2

- Start: leave first item (9) in place
- 1st pass: 5 inserted → 5, 9, 4, 15, 3, 8, 11, 2
- 2nd pass: 4 inserted → 4, 5, 9, 15, 3, 8, 11, 2
- 3rd pass: 15 stays in place → 4, 5, 9, 15, 3, 8, 11, 2
- 4th pass: 3 inserted → 3, 4, 5, 9, 15, 8, 11, 2
- 5th pass: 8 inserted → 3, 4, 5, 8, 9, 15, 11, 2
- 6th pass: 11 inserted → 3, 4, 5, 8, 9, 11, 15, 2
- 7th pass: 2 inserted → 2, 3, 4, 5, 8, 9, 11, 15

5th pass detail (inserting 8):

- 8 is removed temporarily: 3, 4, 5, 9, 15, _, 11, 2
- 15 > 8 so 15 shifts right: 3, 4, 5, 9, _, 15, 11, 2
- 9 > 8 so 9 shifts right: 3, 4, 5, _, 9, 15, 11, 2
- 5 < 8 so 8 is inserted: 3, 4, 5, 8, 9, 15, 11, 2

### Merge Sort

Merge sort is a two-stage sort:

- Stage 1: The list is successively divided in half, forming two sublists, until each sublist is of length one
- Stage 2: Each pair of sublists is repeatedly merged to produce new sorted sublists until there is only one sublist remaining — this is the sorted list

Example — sorting: 42, 31, 12, 3, 37, 18, 29, 47

Stage 1 — splitting:

- 42, 31, 12, 3, 37, 18, 29, 47
- 42, 31, 12, 3  |  37, 18, 29, 47
- 42, 31  |  12, 3  |  37, 18  |  29, 47
- 42  |  31  |  12  |  3  |  37  |  18  |  29  |  47

Stage 2 — merging in sorted order:

- 31, 42  |  3, 12  |  18, 37  |  29, 47
- 3, 12, 31, 42  |  18, 29, 37, 47
- 3, 12, 18, 29, 31, 37, 42, 47

### 6.4 – Developing Algorithms Using Flowcharts

A flowchart is a diagram that uses standard symbols to show the flow of data, processing and input/output in a program or task.

Standard flowchart symbols:

- Oval/rounded rectangle: Terminal symbol — START or END of a program or subprogram
- Rectangle: Process symbol — e.g. count = count + 1 or total = (a + b) + 3
- Parallelogram: Input/Output symbol — e.g. number = input("Enter number:") or print("Total = ", total)
- Diamond: Decision symbol — accepts only Yes or No answers — e.g. "Is number less than 0?"
- Arrow/flow line: Shows direction of program flow
- Rectangle with double vertical lines: Subprogram symbol

Example 6 — flowchart logic:

- number is continuously multiplied by 2 and added to sum on each iteration
- The loop continues until sum exceeds 29
- When sum > 29, sum is output and the program ends

Pseudocode equivalent:

```
sum = 0
number = 1
while sum <= 29
    number = number * 2
    sum = sum + number
endwhile
print sum
```

### 6.5 – Developing Algorithms Using Pseudocode

Pseudocode uses programming-style constructs but is not a real programming language. You describe the steps needed without worrying about exact syntax.

There are three basic programming constructs used in pseudocode:

- sequence
- selection
- iteration

---

### Sequence

Sequence is writing steps in the order they need to happen.

Plain pseudocode example:

```
Input the product price
Input the quantity
total = quantity * price
Output "Total price is " + total
```

OCR Exam Reference Language equivalent:

```
productPrice = input("Please enter product price")
quantity = input("Please enter quantity")
total = quantity * price
print("Total price is ", total)
```

Note: productPrice = input("Please enter product price") means — display the prompt on screen, and when the user enters a value, assign it to the variable productPrice.

---

### Selection

Selection uses if...then...else statements to choose between options.

if...then...else:

```
if x <= 10 then
    z = z + 10
else
    z = z - 10
    y = y + 1
endif
```

if...then (no else):

```
if gameWon == True then
    (instructions here)
endif
```

if...elseif...else (multiple conditions):

```
if menuChoice == 1 then
    display rules
elseif menuChoice == 2 then
    play game
elseif menuChoice == 3 then
    exit
endif
```

```

    case "3": print("exit")

```

### Iteration

There are three basic iteration (loop) constructs:

for loop — executes a specific number of times. The step command allows counting in multiples or backwards (e.g. for i = 10 to 0 step -1):

```
for count = 1 to 10
    print(count * 3)
next count
```

Note: next count is used in algorithms to make clear that count is incremented each time around the loop.

do...until loop — condition is checked at the END of the loop, so it always executes at least once:

```
count = 1
do
    print(count * 3)
    count = count + 1
until count == 10
```

while loop — condition is checked at the START of the loop, so it executes zero or more times. Useful when you need to check a condition before executing (e.g. checking a file is not empty before reading):

```
while NOT customerfile.endOfFile()
    write count * 3
    count = count + 1
endwhile
```

Note: customerfile.endOfFile() determines whether the end of the file called customerFile has been reached.

### 6.6 – Interpret, Correct or Complete Algorithms

A trace table shows how variable values change as an algorithm runs. It is used to:

- Interpret what an algorithm is doing
- Find and correct errors in an algorithm
- Dry run an algorithm before writing program code

Meaningful variable names are a key part of understanding and interpreting the function of an algorithm.

Example 9 — trace table for binary-to-denary conversion:

Algorithm:

```
total = 0
conversion = 128
for count = 1 to 8
    digit = input("Please enter a digit")
    value = digit * conversion
    total = total + value
    conversion = conversion / 2
next count
print(total)
```

Input digits: 1, 1, 0, 1, 1, 0, 1, 0

Trace table:

```
Count | Digit | Value | Total | Conversion
  -   |   -   |   -   |   0   |   128
  1   |   1   |  128  |  128  |    64
  2   |   1   |   64  |  192  |    32
  3   |   0   |    0  |  192  |    16
  4   |   1   |   16  |  208  |     8
  5   |   1   |    8  |  216  |     4
  6   |   0   |    0  |  216  |     2
  7   |   1   |    2  |  218  |     1
  8   |   0   |    0  |  218  |     -
OUTPUT: 218
```

This algorithm converts an 8-bit binary number into a base 10 (denary) number.

Example 10 — identifying an error using a trace table:

Original algorithm (y = 500 / n) — trace with inputs 10, 1000, -2, 0:

```
n=10    → y=50
n=1000  → y=0.5
n=-2    → y=-250
n=0     → ERROR (division by zero — program would crash)
```

Dry running reveals the error. The corrected algorithm adds a check:

```
n = input
if n = 0 then
    print("Enter a non-zero value: ")
else
    y = 500 / n
    print y
endif
```

### Completion of Algorithms

Algorithms may be incomplete and need extra steps inserted to work correctly.

Example — temperature analysis (first attempt, incomplete):

```
10  for count = 1 to 100
20      temperature = input("Enter temperature")
30      if temperature >= 0 then
40          t1ZeroOrMore = t1ZeroOrMore + 1
50      else
60          t2LessThanZero = t2LessThanZero + 1
70      endif
80  next count
90  print(t1ZeroOrMore, t2LessThanZero)
```

Missing parts identified:

- Variables not initialised to 0 before the loop
- No summation of all 100 temperatures (needed for mean)
- Mean temperature not calculated or output

Completed algorithm:

```
5   t1ZeroOrMore = 0
6   t2LessThanZero = 0
7   totalTemp = 0
10  for count = 1 to 100
20      temperature = input("Enter temperature")
30      if temperature >= 0 then
40          t1ZeroOrMore = t1ZeroOrMore + 1
50      else
60          t2LessThanZero = t2LessThanZero + 1
70      endif
75      totalTemp = totalTemp + temperature
80  next count
90  print(t1ZeroOrMore, t2LessThanZero)
95  meanTemperature = totalTemp/100
96  print meanTemperature
```
