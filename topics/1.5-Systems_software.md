### Operating System (OS)

The operating system is a group of programs that manages the computer's resources.

Functions of the operating system:

- Providing a user interface
- Memory management
- Peripheral management
- User management
- File management

---

### Providing a User Interface

The user interface is the way in which we interact with computer hardware.

- WIMP stands for Windows, Icons, Menus and Pointers
- WIMP/GUI interfaces: used on desktop computers — clicking icons, scrolling, typing into forms
- Mobile/tablet interfaces: allow swiping and sensing orientation changes; include hardware controls for specific functions (e.g. camera, sound)
- Embedded systems (e.g. cars, central heating): require specialised interfaces provided by the OS
- Command-line interface (CLI): text-only; user types commands directly; no mouse or menus
- Before GUI, PCs used CLI; advanced users still prefer CLI for direct control
- MS Windows still provides access to a command-line interface

---

### Memory Management

When a program is running it must be in the computer's main memory.

- The OS manages where in memory each program and its data are stored
- Most computers hold several programs in memory simultaneously so users can switch between applications
- A computer that switches rapidly between several tasks is called a multi-tasking computer
- When a program starts, the memory manager allocates blocks of free space in main memory
- Memory is also allocated for any data files opened by the program
- When a program finishes or no longer needs data, the memory management system frees that space for reuse

---

### Peripheral Management

Peripherals are any computer hardware components that are not part of the CPU.

- Includes input, output, and storage devices
- Storage devices (e.g. hard disks) are peripherals even though they may be inside the casing — they are outside the CPU (processor and main memory)
- The OS manages peripherals using device drivers
- A device driver is a small program that acts as an interface between the computer and the device
- Different drivers exist for different operating systems (e.g. an HP printer needs different drivers for PC and Mac)
- When a print instruction is given, peripheral management controls the data transfer from memory to the device driver; the user can continue working meanwhile
- Buffering: used to compensate for differences in speed between the rate data is received and the rate it can be printed or streamed
- A buffer is an area of RAM controlled by the OS or peripheral management system
- Data to be printed is copied to the print buffer; the print management system retrieves and sends it to the printer
- If the buffer fills before printing is complete, it is refilled as space becomes available

---

### User Management

User management enables a network administrator to:

- Allocate accounts and set different access rights for different users
- Identify all users currently on the network
- Manually log out users
- Monitor when and for how long each user is logged in over a period of time
- Set default logout times based on inactivity

---

### File Management

A file system is normally organised into folders and subfolders for easy user navigation and usage.

File management systems:

- Enable a user to create, name, save, modify, copy, delete and move files
- Enable a user to search for a particular file
- Keep track of the location of files on disk or other storage device so they can be retrieved when needed
- Keep track of free space on disk where files can be stored
- Enable users to restore deleted files
- Prevent conflicts when two users on a network attempt to modify the same file
- Maintain access rights to files

---

### Utility Software

The operating system controls and manages the computer system, but most operating systems also include utilities.

- Utilities are not essential for the computer to work
- They make it easier to perform particular tasks or provide housekeeping functionality
- Examples:
  - Security utilities (e.g. encryption software)
  - Disk organisation utilities (organise files into folders, tidy up disk)
  - Data compression utilities

---

### Encryption Software

Encryption software keeps sensitive data safe from cyber-criminals or unauthorised access.

- Protects backup copies of data stored on offline storage (e.g. flash memory sticks, laptops)
- Encrypting data ensures it cannot be interpreted by those without the key
- The encryption process uses an algorithm and a key to transform plaintext into ciphertext
- Plaintext: the original, readable data
- Ciphertext: the encrypted, unreadable form of the data
- To decode (decrypt), both the algorithm and the key must be known

---

### Defragmentation Software

Files on a hard disk are stored in blocks wherever there is space.

- Large files may be split into segments stored in different locations — this is fragmentation
- Over time, thousands of files become fragmented across the disk
- Fragmentation slows down file access because the OS must track all segments
- The disk defragmenter moves separate file parts so they are stored together
- It also groups all free disk space together so new files can be stored in one place
- This optimises disk performance
- Note: SSDs do not need defragmenting — doing so brings no benefit and may shorten the drive's life

---

### Compression Software

Compression software (e.g. WinZip) reduces the size of a file.

Two types of compression:

- Lossy compression: some data is permanently lost; used for photos/audio/video where small quality loss is undetectable on screen
- Lossless compression: the file can be restored exactly; used for text files where accuracy is essential

Advantages of compression:

- Compressed files can be transmitted much more quickly over the internet
- Allows files to be emailed where attachment size limits apply
- Multiple files can be compressed into a single zip file for easier transmission, storage, or movement

### Misconceptions

- misconception: The operating system and utility software are the same thing
  reality: The OS is essential software that manages computer resources; utilities are additional programs that perform housekeeping tasks. Utilities are not essential for the computer to function.
  trigger: When students list utilities as OS functions or confuse the two
  correction_prompt: Be careful not to mix up the OS and utility software. The OS is essential — the computer cannot work without it. Utilities are extras that make it easier to manage or maintain the system.

- misconception: Defragmentation improves the performance of all storage devices
  reality: Defragmentation only benefits HDDs. SSDs work differently and do not suffer from fragmentation in the same way — defragging an SSD brings no benefit and may reduce its lifespan.
  trigger: When a student states that all storage devices should be defragmented
  correction_prompt: Defragmentation only applies to hard disk drives (HDDs). SSDs should never be defragmented — it does not improve performance and can shorten the drive's life.

- misconception: Lossy compression can always be reversed to recover the original file
  reality: Lossy compression permanently discards some data. Only lossless compression allows exact restoration of the original file.
  trigger: When a student states that compressed files can always be restored exactly
  correction_prompt: There are two types of compression. Lossless compression can be fully reversed. Lossy compression cannot — some data is permanently lost. The type used depends on whether exact restoration matters.

- misconception: A buffer is the same as main memory (RAM)
  reality: A buffer is a specific area of RAM reserved by the OS for managing data transfers to/from peripherals. It is not the same as general-purpose RAM used by programs.
  trigger: When students describe buffering without distinguishing it from general memory use
  correction_prompt: A buffer is not simply RAM — it is a dedicated area of RAM used specifically to manage the speed difference between the CPU and a peripheral device such as a printer.

- When explaining defragmentation, proactively mention that SSDs should not be defragmented
- When explaining compression, always contrast lossy and lossless and give an example use case for each
- Use consistent sentence structures when describing OS functions
